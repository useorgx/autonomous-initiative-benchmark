# OrgX Acceptance Schema v1 — production-acceptance contracts (per domain)

**Date:** 2026-06-23 · authored by a 36-agent adversarially-hardened workflow (2.8M tokens). These are the dual-use contracts: the benchmark validator AND the product-facing acceptance gate run the SAME schema. Expected single-pass frontier pass rates are 5-25% — the 100x bar.

The shared check primitives (15): selfReportRecomputeMatch, contrastRatioWCAG, thresholdTableLookup, deterministicSimulationRecompute, canonicalSourceTrace, typedSchemaConformance, forbiddenLiteralLint, referenceGraphResolve, antiCollapseDistinctness, perceptibleDelta, uniqueNonGenericCue, coverageSuperset, worstCaseResolution, diffAgainstLiveRepo, labelJustifiedByResolvedAttr.

## UI color palette / design tokens — production DTCG token PR merged to the shared OrgX component library ("Luminous Glass" palette). The deliverable is a complete, machine-checkable design-token set: b

**Ship gate:** SHIP ONLY IF: (1) all blocker checks pass — C01 (AA body both modes), C02 (non-text/focus-ring 3:1), C03 (large-text, with size-justification), C05 (brand trace to a DECLARED canonical source), C06 (DTCG typed), C07 (semantic alias-only, no dangling/cyclic), C11 (focus appearance incl. contrast vs focused fill, recomputed), C13 (semver diff vs live token files), C16 (translucent worst-case backdrop), C17 (pairs ⊇ machine-extracted manifest), C18 (build-output checksum determinism); AND (2) ALL self-reported numbers (measured.wcagRatio within 0.05, measured.apcaLc within 1.0, colorBlindReport deltaE within 0.5) match independent recompute; AND (3) at most ONE major check (C04, C08, C09, C10, C12, C14, C15, C19) may be in a documented, time-boxed waiver with an owner and expiry — two or more open major failures block; AND (4) zero minor (C20) regressions versus the previous token package version. Any blocker fail, any recompute mismatch, or a missing canonical-source declaration is an automatic NO-SHIP regardless of other passes.

**Expected frontier pass rate:** Estimated 8–15% single-pass on the FULL gate. Reasoning: a frontier model reliably nails the structural/standards checks (C06, C07, C10, C20) and the easy contrast math on opaque pairs (C01/C03 light mode) — those alone might score ~70% in isolation, which is exactly why the original gate was trivially 100%. The pass rate collapses on the holes the reviews exposed: (a) C17 requires the artifact's pairs[] to be a SUPERSET of a manifest the model cannot see at authoring time (the checker extracts it from source), so any under-declaration of inconvenient pairs — lime-on-white CTA text, ink-soft on glass, status text on its own gradient — fails; models almost always under-declare. (b) C16 forces worst-case translucent backdrops; a model that picks the flattering #02040a backdrop fails, and OrgX's palette is overwhelmingly rgba-on-glass/gradient, so most contrast claims become genuinely hard and several real OrgX tokens (e.g. warm warning text on its 0.12 surface) plausibly FAIL the math outright, meaning even a faithful artifact can be blocked until colors are adjusted. (c) C05 + brandCore force resolving a real repo contradiction (PALETTE.md #BFFF00/#14B8A6/#6366F1 + glass 0.05 + dark-first vs tokens.json #D4ED31/#00C9A7/#7C7CFF + glass 0.025 + light-first); a model that doesn't notice and silently traces to a blend fails C05. (d) C13 diffs against live token names in design-tokens.ts/globals.css — a model treating this as greenfield ('breaking:[]') fails because real consumer tokens exist. (e) The recompute-match clauses (wcag within 0.05, apca within 1.0, CVD deltaE within 0.5) punish any fabricated or sloppily-computed number, and APCA via the published lookup matrix (C04) is rarely implemented correctly. (f) C19 blocks the tempting escape hatch of shipping a flat, perfectly-accessible-but-off-brand palette. Clearing every blocker simultaneously, with all self-reported numbers surviving independent recompute, on the first attempt is the conjunction of ~11 independent hard conditions — hence well below 20%, centered around 10%.

### Enriched artifact schema
- `tokens[].$type` (string enum (color|dimension|fontFamily|duration|cubicBezier|shadow|gradient), required) — DTCG compliance — every token typed so it is machine-validatable and transformable. gradient added because OrgX's identity (energyGradient/auraGradient/status backgrounds) is gradient-typed, not flat color; untyped gradients cannot be checked.
- `tokens[].$value` (string|object (hex/rgb-triplet for color; reference {token} for semantic; ordered stop array for gradient), required) — Actual value. Semantic tokens MUST be aliases ({color.brand.teal}) not raw hex so brand traceability is enforceable; gradients carry ordered stops so glow/sweep fidelity is checkable.
- `tokens[].$description` (string (>= 12 chars, unique among same-layer tokens), required) — Human-readable role; uniqueness requirement detects copy-paste duplicate roles (the alias-laundering monotone-palette dodge).
- `tokens[].layer` (string enum (primitive|semantic|component), required) — Three-tier discipline. Lets the checker enforce 'no raw hex in semantic/component layers' and 'semantic must not alias another semantic of a different role family' — kills alias-laundering.
- `tokens[].modes` (object {light: value, dark: value}, required) — Both themes ship here. Repo is dark-first (--bg #02040a) but tokens.json is light-first (#F0FFFE) — the artifact must reconcile, and dark cannot be a trivial transform of light (checked, see C14).
- `tokens[].states` (object {default,hover,active,focus,disabled} (required only when layer=component AND role in interactiveRoles), required) — Interactive roles (cta/button/link/input) must define every state with PERCEPTIBLY different values (checked, see C09), not five near-identical numbers.
- `pairs[]` (array of {fgToken, bgToken, role: body|large|nonText|disabled|focusRing, mode, effectiveBg}, required) — Contrast is a property of fg-on-bg PAIRS. effectiveBg is the REQUIRED resolved opaque backdrop after compositing any translucent/gradient layers (see assumptions + C02/C16), so contrast is computable and not a guess.
- `pairs[].measured` (object {wcagRatio: number, apcaLc: number}, required) — Self-reported computed values the checker recomputes independently and must match within tolerance; exposes fabricated numbers.
- `renderedPairsManifest` (object {sourceFiles: string[], extractedPairs: [{fgToken,bgToken,role,mode}], coverageHash}, required) — Closes the under-declaration hole. The checker independently extracts every fg/bg co-occurrence from the cited component/CSS sources; pairs[] must be a SUPERSET of this machine-extracted manifest (see C17). The author cannot omit the inconvenient pairs.
- `statusTokens[].nonColorCue` (object {icon: string (from allowed lucide set), label: string, shape: string}, required) — SC 1.4.1. icon must be from an enumerated allowed set AND be unique per status; label must be unique per status and != generic ('Status','Info') — kills the icon:'Circle'/label:'Status'-on-everything dodge.
- `focusIndicator` (object {color, thicknessPx, offsetPx, contrastVsFocused, contrastVsAdjacent, appliedToRoles: string[]}, required) — SC 2.4.7/2.4.11/2.4.13. Focus is first-class with measurable contrast against BOTH the focused element fill and the adjacent surface, evaluated per role it is applied to (so lime-on-teal-CTA is actually checked).
- `colorBlindReport` (object {deuteranopia,protanopia,tritanopia: {minPairwiseDeltaE2000, simulatedHexes: object}}, required) — Status palette must stay distinguishable under CVD simulation. simulatedHexes lets the checker RE-RUN the Brettel/Viénot simulation + CIEDE2000 itself rather than trusting a number; threshold is fixed in C12.
- `claims[]` (array of {statement, standard, scope: {role,mode,pairRefs:int[]}, provenance:{tool,version}, computedValue}, required) — Every accessibility/brand claim cites the standard, the EXACT pairs it covers (scope.pairRefs), and the tool/version. The checker verifies the claim's quantifier matches its scope (e.g. 'all text AAA' must reference every body/large pair) — kills overclaiming (see C15).
- `assumptions[]` (array of {assumption, appliesToTokens: string[], assumedBackdrop: value, rationale}, required) — Translucent/gradient backdrop assumptions are explicit and PER-TOKEN. The checker rejects a single global flattering backdrop and forces the assumed backdrop to equal the WORST-CASE real surface a token renders over (see C16).
- `version + changelog` (object {semver, breaking: token[], added: token[], removed: token[], renamed: [{from,to}]}, required) — Token packages are a consumed contract. The checker diffs the artifact against the live token names in design-tokens.ts/globals.css; any name present live but absent/renamed here MUST appear in breaking/removed/renamed with a MAJOR bump (see C13) — greenfield 'breaking:[]' is rejected if real consumers exist.
- `buildOutputs` (object {css: {path,checksum}, ts: {path,checksum}, source: {path,checksum}, generator: string}, required) — PALETTE.md mandates CSS + TS stay in sync from one source. Both generated outputs must be byte-derivable from the source token file; the checker regenerates and compares checksums (see C18).
- `brandCore` (object {canonicalSource: 'PALETTE.md'|'tokens.json', lime,teal,iris,ink,inkSoft: hex, reconciliationNote: string}, required) — The two repo sources DISAGREE (PALETTE.md #BFFF00/#14B8A6/#6366F1 vs tokens.json #D4ED31/#00C9A7/#7C7CFF; glass 0.05 vs 0.025; dark-first vs light-first). The artifact must DECLARE which file is canonical and reconcile; the checker then traces against that single declared source (see C05).
- `identityFidelity` (object {gradientTokens: token[], glowTokens: token[], auraTokens: token[], sweepOrderCheck: object}, required) — 'Luminous Glass' identity is the gradient/glow/aura system. Closes the accessible-but-dead-flat-palette bypass: the artifact must include the required count of gradient/glow/aura tokens whose stops trace to the brand sweep in canonical order (see C19).

### Checks (20)
- **C01-aa-body-contrast** [blocker, accessibility] — Every pair with role=body must meet WCAG 2.2 SC 1.4.3 normal-text contrast >= 4.5:1 in BOTH light and dark mode, computed on resolved opaque effectiveBg.
  - compute: For each pairs[] where role=body: resolve fg and effectiveBg to opaque sRGB (alpha-composite translucent/gradient per C16 onto declared assumedBackdrop, gradient uses worst-case stop). ratio = (Ll+0.05)/(Ls+0.05) using relative luminance (linearize each channel: c<=0.03928? c/12.92 : ((c+0.055)/1.055)^2.4; L=0.2126R+0.7152G+0.0722B). PASS if ratio>=4.5 for light AND dark. Recompute independently and require |computed-measured.wcagRatio|<=0.05.
  - threshold: 100% of body pairs >= 4.5:1 in both modes; self-reported wcagRatio matches recompute within 0.05
  - gameability: Could pass by declaring only easy body pairs — closed by C17 superset check forcing all machine-extracted body pairs to be present; could pass with a flattering backdrop — closed by C16 worst-case rule.
- **C02-nontext-contrast** [blocker, accessibility] — SC 1.4.11 — every pair role in {nonText,focusRing} (borders, focus rings, icon strokes, chart strokes, status dots) >= 3:1 against adjacent color, both modes.
  - compute: Same luminance ratio as C01 over pairs[] where role in {nonText,focusRing}. For focusRing, evaluate BOTH contrastVsFocused and contrastVsAdjacent. PASS if all >=3.0 in both modes.
  - threshold: 100% of nonText/focusRing pairs >= 3.0:1 in both modes
  - gameability: focusRing previously unchecked against the element it surrounds — now requires contrastVsFocused>=3 so lime-on-teal-CTA must actually pass.
- **C03-large-text-contrast** [blocker, accessibility] — SC 1.4.3 — large text (>=24px or >=18.66px bold) pairs role=large >= 3:1 both modes.
  - compute: Luminance ratio over role=large pairs. Cross-check: for each large pair, assert the referenced typography token resolves to size>=24px OR (>=18.66px AND weight>=700) from tokens.json typography.scale; if not, FAIL (mislabeled large text).
  - threshold: 100% role=large pairs >= 3.0:1 AND size/weight justifies 'large'
  - gameability: Closes 'call body text large to dodge 4.5' — the size/weight cross-check from the real type scale blocks relabeling.
- **C04-apca-lookup** [major, accessibility] — APCA Lc must meet the size/weight-specific minimum from the APCA font-lookup matrix, not a flat Lc>=75 (the reviewer's correct objection).
  - compute: For each body/large pair: compute APCA Lc (sRGB->Y with exponent 2.4, Ynorm, soft-clamp black, polarity-aware SAPC per APCA-W3 0.1.9 constants), take abs(Lc). Look up the required Lc from a bundled APCA font-size/weight table keyed by the pair's resolved px size and weight (e.g. 16px/400 needs Lc>=90 for fluent body... use the published APCA bronze table). PASS if abs(Lc) >= tableMinimum for that size/weight. Require |computed-measured.apcaLc|<=1.0.
  - threshold: 100% of body/large pairs meet the matrix-derived Lc minimum for their size/weight; reported apcaLc within 1.0 of recompute
  - gameability: Flat Lc>=75 was a fake contract — replaced by the deterministic published lookup table indexed on the real type token; no judgment, the table is data.
- **C05-brand-trace-canonical** [blocker, brand] — All primitive brand stops must trace to the SINGLE declared canonical source (brandCore.canonicalSource); the two repo sources disagree and the artifact must pick one and match it exactly.
  - compute: Read brandCore.canonicalSource. If 'PALETTE.md', required lime/teal/iris/ink/inkSoft = #BFFF00/#14B8A6/#6366F1/#F2F7FF/#A5B2D0. If 'tokens.json', required lemon/aqua/iris = #D4ED31/#00C9A7/#7C7CFF (and ink from tokens.json text.primary #0A1F1F). Parse the chosen file at the cited path, extract those hexes, and assert brandCore values == file values (exact, case-insensitive). FAIL if canonicalSource unset or values mismatch the chosen file.
  - threshold: brandCore matches the declared canonical file's values exactly; canonicalSource is set
  - gameability: Reviewer flagged the two sources conflict so a checker can't pick — fixed by making the ARTIFACT declare canonical, then deterministically matching that one file. Tracing to neither now fails (canonicalSource must resolve).
- **C06-dtcg-typed** [blocker, standards] — DTCG conformance — every token has $type from the allowed enum, $value, $description, and layer.
  - compute: For every token: assert presence of $type in {color,dimension,fontFamily,duration,cubicBezier,shadow,gradient}; $value present; $description length>=12; layer in {primitive,semantic,component}. Count violations.
  - threshold: 0 tokens missing/invalid $type/$value/$description/layer
  - gameability: Structural; pairs with C07 so typing isn't satisfiable by stub values.
- **C07-semantic-alias-only** [blocker, standards] — No raw hex/rgb literals in semantic or component layers — they must be {token} aliases to primitives.
  - compute: For tokens where layer in {semantic,component}: assert $value (and every gradient stop, every state value) is a reference string matching /^\{[a-zA-Z0-9._-]+\}$/ resolving to an existing token id; FAIL on any raw hex/rgb/hsl literal. Resolve alias graph; FAIL on cycles or dangling refs.
  - threshold: 0 raw color literals in semantic/component layers; 0 dangling/cyclic aliases
  - gameability: Alone this rewards mechanical referencing (monotone dodge) — neutralized by C08 role-family rule + C03/C19 role differentiation.
- **C08-alias-role-family** [major, correctness] — Defeat alias-laundering: a semantic token must alias a primitive whose hue role matches the semantic token's role family (a 'positive/success' token cannot alias the same primitive as a 'cta' token if their declared roles differ), and the set of distinct primitives referenced by distinct semantic role families must be >= a minimum.
  - compute: Build map: for each role family in {ctaFill, positive, warning, danger, info, secondaryGradient, mutedText}, collect the primitive(s) its semantic tokens resolve to. Assert: (a) distinct primitives across these families >= 6; (b) no two families with conflicting hue intent (e.g. danger vs positive) resolve to the SAME primitive; hue conflict computed via HSL hue distance < 25deg on resolved primitive == 'same hue'. FAIL otherwise.
  - threshold: >=6 distinct primitive hues across role families; 0 cross-family hue collisions for opposed roles
  - gameability: Directly kills 'alias everything to teal/lime/iris' — a monotone palette collapses distinct-primitive count below 6 and collides opposed roles.
- **C09-state-perceptible-delta** [major, completeness] — Interactive component tokens must define all 5 states with PERCEPTIBLY different values (not five near-identical numbers), and disabled must read as lower-emphasis not as a contrast failure.
  - compute: For each component token with role in interactiveRoles: assert states has default/hover/active/focus/disabled. Resolve each to opaque sRGB. Require CIEDE2000(default,hover)>=2.0 AND CIEDE2000(default,active)>=2.0 (perceptible). For disabled: require its body-text pair APCA abs(Lc) is in [30,60] (legibly muted, not invisible and not full-strength) AND deltaE(default,disabled)>=3.0. FAIL on any unmet.
  - threshold: All 5 states present; hover/active deltaE2000>=2.0 vs default; disabled Lc in [30,60] and distinct
  - gameability: Closes 'emit five near-identical values'; deltaE2000>=2.0 is the established JND threshold (not a taste pick), disabled band keeps it from being a stealth contrast-fail.
- **C10-use-of-color** [major, accessibility] — SC 1.4.1 — every status token carries a non-color cue with a UNIQUE, non-generic icon AND label AND shape.
  - compute: Allowed icon set = lucide names present in tokens.json statusIndicators (e.g. MinusCircle,AlertCircle,XCircle,Activity,CheckCircle,CircleDot). For statusTokens[]: assert icon in allowed set; assert icon values are pairwise-unique across statuses; assert label pairwise-unique and not in banned generic set {'Status','State','Info','Item','Circle',''}; assert shape present and pairwise-unique. FAIL otherwise.
  - threshold: All statuses: icon from allowed set + unique; label unique and non-generic; shape unique
  - gameability: Reviewer's icon:'Circle'/label:'Status' clone dodge is blocked by pairwise-uniqueness + banned-generic list; can't satisfy schema with identical cues.
- **C11-focus-appearance** [blocker, accessibility] — SC 2.4.7/2.4.11/2.4.13 — focus indicator >=2px thick, has offset, contrast >=3:1 vs BOTH focused fill and adjacent surface, applied to every interactive role.
  - compute: Assert focusIndicator.thicknessPx>=2 AND offsetPx>=0 AND contrastVsFocused>=3.0 AND contrastVsAdjacent>=3.0 (recompute both ratios from resolved colors, not trusting reported). Assert appliedToRoles superset-equals interactiveRoles found in component tokens. Recompute contrast independently; FAIL on mismatch>0.05 or any threshold miss.
  - threshold: thickness>=2px, both contrasts>=3:1 (recomputed), applied to 100% of interactive roles
  - gameability: Recomputing contrast vs the actual CTA fill the ring surrounds blocks the lime-on-teal-invisible-ring case; appliedToRoles coverage blocks declaring focus for one harmless role only.
- **C12-cvd-distinguishable** [major, accessibility] — Status palette stays distinguishable under deuteranopia/protanopia/tritanopia — checker RE-SIMULATES and applies a FIXED threshold (reviewer correctly noted threshold was unspecified).
  - compute: Independently simulate each status dot color under all three CVD types using the Viénot 1999 / Brettel matrices (deterministic, published). Compute pairwise CIEDE2000 among simulated status colors per CVD type. THRESHOLD fixed at minPairwiseDeltaE2000 >= 11.0 (this is the contract's chosen, stated cutoff — pickable because the contract OWNS it, removing the judgment). Require artifact-reported colorBlindReport.minPairwiseDeltaE2000 within 0.5 of recompute, and recompute >= 11.0 for all three.
  - threshold: recomputed min pairwise deltaE2000 >= 11.0 under all 3 CVD types; reported within 0.5 of recompute
  - gameability: Threshold is now explicit and owned by the contract (11.0), simulation is recomputed from published matrices, so neither the cutoff nor the number is a hidden judge call.
- **C13-semver-breaking-diff** [blocker, reproducibility] — Changelog must reflect a REAL diff against the live token names in the repo — greenfield 'breaking:[]' is rejected if existing consumers reference removed/renamed tokens.
  - compute: Parse live token identifiers from lib/design-tokens.ts and the --var names in app/globals.css (regex over export object keys + /--[a-z0-9-]+:/). liveSet = those names. artifactSet = token ids/names produced. removedOrRenamed = liveSet not present in artifactSet (after applying changelog.renamed map). For each such name, assert it appears in changelog.removed or changelog.renamed.from. If removedOrRenamed nonempty, assert semver MAJOR bump vs tokens.json current version (1.4.0 -> >=2.0.0). FAIL otherwise.
  - threshold: Every live token name absent from the artifact is declared in removed/renamed; MAJOR bump iff any breaking; 0 silent drops
  - gameability: Closes greenfield-changelog gaming by diffing against the ACTUAL live token files, not the author's self-report.
- **C14-dark-not-trivial-transform** [major, completeness] — Dark mode must be a genuine theme, not light auto-inverted/uniformly darkened; and dark glows/ink must be legible on dark bg.
  - compute: For all color tokens: collect (light,dark) pairs. (a) Reject trivial transform: fit best single affine lightness map L_dark = a*L_light + b across tokens; if residual RMSE < 2.0 (i.e. dark is essentially a uniform function of light) FAIL. (b) Legibility: in dark mode, every body pair must still pass C01 (>=4.5) and every glow/aura token must have deltaE2000 vs --bg(#02040a) >= 10 (visible on near-black). FAIL otherwise.
  - threshold: affine-fit residual RMSE >= 2.0 (non-trivial) AND dark body pairs >=4.5:1 AND glow vs #02040a deltaE>=10
  - gameability: Blocks 'dark = light darkened/duplicated': a uniform transform produces tiny residual and fails; glow-visibility check catches glows that vanish on #02040a.
- **C15-claim-scope-match** [major, correctness] — Each claim's quantifier must match the pairs it actually covers (no 'meets AAA for all text' when only body is computed).
  - compute: For each claim: parse quantifier (all|every|some|N). Resolve scope.pairRefs to actual pairs. If quantifier in {all,every} for a role class (text/large/nonText), assert scope.pairRefs covers EVERY pair of that class in pairs[] (set equality). Recompute each referenced pair against the claim's standard threshold (e.g. AAA text=7:1, AAA large=4.5:1) and assert claim.computedValue matches and passes. FAIL on coverage gap or failing pair under an 'all' claim.
  - threshold: Universal claims cover 100% of their role class; every referenced pair recomputes to satisfy the cited standard
  - gameability: Reviewer's 'statement says AAA-all but only body computed' is caught deterministically by comparing the claim's quantifier-set against the full pair class — no NL judgment, just set coverage + recompute.
- **C16-translucent-worstcase-backdrop** [blocker, robustness] — Translucent/gradient tokens must be evaluated over their WORST-CASE real backdrop, not one flattering opaque assumption.
  - compute: For every token whose $value is rgba alpha<1 or gradient: candidateBackdrops = the set of surface tokens it actually composites over per renderedPairsManifest (e.g. --bg, --surface-subtle, marketing aura). For each contrast check involving this token, the checker composites the token over EACH candidate backdrop (gradient: evaluate min-contrast stop) and uses the MINIMUM resulting contrast. assumptions[].assumedBackdrop for the token must equal that worst-case backdrop; if the author declared a different (easier) backdrop, FAIL. All C01/C02 checks then use the worst-case composite.
  - threshold: Each translucent/gradient token's contrast graded on min over its real backdrops; assumedBackdrop == worst-case surface
  - gameability: Directly kills the 'assume glass over #02040a everywhere' dodge — the checker derives candidate backdrops from the manifest and forces worst-case, ignoring a self-serving single assumption.
- **C17-pair-coverage-superset** [blocker, completeness] — pairs[] must be a SUPERSET of the machine-extracted rendered-pair manifest — the author cannot omit inconvenient pairs.
  - compute: Independently extract fg/bg co-occurrences: parse component sources + globals.css for text-color-on-background and border-on-surface usages (regex: text token + nearest bg/surface token in same rule/className), plus enumerate every status text on its own statusIndicators.background gradient, and every interactive role default+hover fills. Build extractedPairs. Assert pairs[] (by {fgToken,bgToken,role,mode}) is a superset of extractedPairs; report missing. FAIL if any extracted pair absent. coverageHash of extractedPairs must match renderedPairsManifest.coverageHash.
  - threshold: pairs[] ⊇ machine-extracted manifest (0 missing); coverageHash matches
  - gameability: This is THE fix for the re-created original hole — completeness is no longer author-asserted; the gate has an independent oracle (source extraction) for the pair universe, including status-text-on-its-own-gradient and lime-on-white-CTA.
- **C18-build-output-determinism** [blocker, reproducibility] — CSS and TS outputs must be byte-derivable from the single source token file (PALETTE.md mandate).
  - compute: Run the declared generator on buildOutputs.source; recompute sha256 of emitted CSS and TS. Assert recomputed checksums == buildOutputs.css.checksum and .ts.checksum. Also assert every primitive token value appears in BOTH emitted css (--var) and ts accessor. FAIL on any checksum mismatch or value present in one output but not the other.
  - threshold: Regenerated CSS/TS checksums match declared; 100% primitive values present in both outputs
  - gameability: Closes CSS/TS drift; a hand-edited output that doesn't regenerate from source fails the checksum recompute.
- **C19-identity-fidelity** [major, brand] — 'Luminous Glass' identity must be present and correct: required gradient/glow/aura tokens whose stops trace, in order, to the brand energy sweep — blocking the accessible-but-dead flat palette.
  - compute: Assert identityFidelity.gradientTokens length>=3, glowTokens length>=4, auraTokens length>=1. For the primary energy gradient: resolve stops to primitives; assert ordered hue sequence follows lemon/lime -> aqua/teal -> cyan (canonical sweep from canonical source: hue(stop0) in lime band, hue(last) in cyan/teal band, monotonic hue progression). For each glow token: assert it is an rgba/shadow whose base color resolves to a brand or status primitive (not neutral). FAIL if a required gradient/glow/aura is absent or a 'gradient' is a 2-stop neutral->neutral (no brand hue).
  - threshold: >=3 gradient + >=4 glow + >=1 aura tokens; energy gradient stops trace to brand sweep in canonical hue order; glows carry brand/status hue
  - gameability: A flat #1A1A1A-on-#fff palette that aces WCAG/APCA now FAILS C19: it has no brand-hued gradients/glows/auras, so the soulless-but-accessible bypass is closed.
- **C20-no-orphan-or-dup-roles** [minor, naming] — No two semantic/component tokens share an identical resolved value AND an identical $description role (copy-paste duplicate), and no semantic token is orphaned (referenced by nothing and aliasing nothing meaningful).
  - compute: Group tokens by (resolvedOpaqueHex, normalizedDescription); FAIL if any group with >1 token in semantic/component layers (true duplicate role). Build reference graph from aliases + pairs[]; flag semantic tokens neither referenced by any component/pair nor aliasing a primitive as orphans; FAIL if orphanCount>0.
  - threshold: 0 duplicate (value+role) semantic/component tokens; 0 orphaned semantic tokens
  - gameability: Catches padding the token set with redundant aliases to inflate distinct-primitive counts or to look complete; complements C08.

## Analytics query spec (filters + aggregation) merged into the OrgX metrics layer powering a live multi-tenant dashboard. Stack: Postgres (Supabase), append-only event tables (agent_run_trace_events), p

**Ship gate:** BLOCK ship if ANY blocker check fails OR if 2+ major checks fail OR if the artifact is missing any required schema field. Blockers (all must pass): CORR-PERCENTILE-SEMANTICS, CORR-LABEL-STATISTIC-CONSISTENCY, SEC-TENANCY-PARAMETERIZED, SEC-PARAMETERIZED-QUERY, CORR-NULL-ZERO-POLICY-HONORED, REPRO-GOLDEN-ADVERSARIAL, REPRO-EXPECTED-IS-COMPUTED, BRAND-SERIES-PALETTE-TOKENS, BRAND-EMPTY-STATE-COMPONENT. Hard preconditions before any check runs: (1) all 16 required schema fields present and well-typed; (2) reference_evaluator is executable and deterministic (same input -> identical output across two runs); (3) sql_text parses. The gate is fully mechanical: a CI job loads tokens.md/components.md as fixtures, parses sql_text, executes reference_evaluator against golden_rows and every fixture, walks the EXPLAIN JSON, and runs all 18 checks in plain JS. A single major failure is allowed only if it is CONSIST-SEMVER-DETERMINISTIC-ORDER (minor) — any pairing of major failures, or any blocker, stops the merge. Re-running on identical inputs must yield an identical pass/fail vector (no LLM judge anywhere in the path).

**Expected frontier pass rate:** 15-25% single-pass. Reasoning: a frontier model reliably nails the parameterized-query, percentile_cont usage, UTC/half-open, and semver checks (the 'obvious' correctness items) — that clears maybe 6-7 of 18. The kill zone is the interlocked reproducibility + brand trio that the draft left gameable: (a) REPRO-GOLDEN-ADVERSARIAL demands a >=20-row golden set with ties, NULLs, stddev>0, AND p50!=p95 — models default to clean uniform fixtures, so first-pass they ship a degenerate set and fail; (b) REPRO-EXPECTED-IS-COMPUTED requires a separately-executable reference_evaluator whose output deepEquals every hand-listed expected value — any arithmetic drift between the SQL semantics and the JS reimplementation fails it, and models rarely produce two independently-correct implementations on the first try; (c) BRAND-SERIES-PALETTE-TOKENS requires exact resolution against the real tokens.md RGB triplets plus the correct per-widget primary_rgb — without having read tokens.md a model invents plausible-but-wrong hex and fails; (d) CORR-NULL-ZERO-POLICY-HONORED + BRAND-EMPTY-STATE-COMPONENT jointly forbid COALESCE(measure,0) while declaring 'distinct', which is the exact instinctive thing a model writes (mirroring the live metrics.ts bug). Each of these four is individually ~40-60% likely to fail first-pass, and they are all blockers, so P(all blockers pass) ≈ 0.6·0.55·0.5·0.6·(other blockers ~0.85^4) ≈ 0.10-0.20. Add the missing-field precondition (16 fields incl. reference_evaluator and dispersion_stats are easy to omit) and realistic single-pass full-gate success lands around 15-25%, well below 100% and concentrated on artifacts that genuinely encode dispersion-aware testing, real token binding, and honest no-data semantics.

### Enriched artifact schema
- `sql_text` (string, required) — The literal parameterized query. Required so injection, NULL-handling, percentile semantics, tenancy filter, and timezone casts are statically checkable from the query body rather than inferred from a result or trusted from metadata.
- `metric_id_and_semver` (string (regex: ^[a-z0-9_.]+@\d+\.\d+\.\d+$), required) — Stable metric key + semver so the dashboard binds to one canonical definition and breaking changes are detectable. Machine-parseable so version-bump rules can be enforced.
- `source_lineage` (array<{table:string, columns:string[], pinned_migration_sha:string, grain:string}>, required) — Every column traced to a source table at a pinned schema SHA + declared grain; enables traceability and lets a check assert every column referenced in sql_text is declared here (no undeclared/ghost columns).
- `filter_predicates` (array<{column:string, op:string, value:string, rationale:string, drops_rows:boolean}>, required) — Each WHERE/HAVING clause enumerated with intent and an explicit drops_rows flag so a reviewer (and a check) can verify no rows are silently dropped and that every WHERE term in sql_text is accounted for.
- `aggregation_spec` (object{measure:string, agg_fn:string, grain:string, window:string, dedup_key:string}, required) — Declares exactly what is aggregated, at what grain, over what window, deduped by what key — so double-counting, wrong-grain, and fan-out join bugs are deterministically catchable against the SQL.
- `measure_label_semantics` (array<{column:string, label:string, statistic:'count'|'sum'|'avg'|'min'|'max'|'p50'|'p95'|'p99'|'rate'|'ratio', sql_expr:string}>, required) — Binds each output column's DISPLAY label to the statistic it actually computes AND the exact sql_expr that produces it. Closes the 'rename p95 to latency_ceiling to dodge the regex' hole: the gate keys off the declared statistic, not the label string, and cross-checks sql_expr.
- `timezone_and_window_policy` (object{tz:'UTC', bucket_boundary:string, half_open_interval:boolean, dst_policy:string}, required) — Live dashboards bucket by time; without explicit UTC + half-open [start,end) policy, edge rows double-count or vanish at bucket/DST seams. Checkable against date_trunc/timestamptz usage in sql_text.
- `null_and_empty_semantics` (object{division_guard:string, empty_result_value:'null'|'absent', null_vs_zero:'distinct'|'merged', coalesce_audit:array<{expr:string, justified:boolean}>}, required) — Distinguishes 'no data' from zero and defines divide-by-zero behavior — the exact class of bug in recordToolUsageMetric (success_rate on 0 calls). coalesce_audit lets a check diff the DECLARED policy against actual COALESCE(...,0) in sql_text.
- `edge_case_fixtures` (array<{name:string, input_rows:object[], expected_output:object[], kind:string, derived_from_golden:boolean}>, required) — Named fixtures covering empty set, single row, ties, NULLs, duplicates, boundary timestamps, late/out-of-order events, DST seam, high cardinality. derived_from_golden + an executable reference forces expected_output to be COMPUTED, not author-asserted (closes the self-graded-fixture hole).
- `tenancy_scope` (object{isolation_column:string, enforced:boolean, bound_as_parameter:boolean}, required) — Dashboard is multi-tenant (workspace_id/org_id); a missing or literal-injected tenant filter leaks cross-customer data — a security + correctness blocker. Checkable: column must appear in sql_text WHERE bound as a parameter, not a literal.
- `performance_budget` (object{explain_analyze_json:object, p95_ms:number, rows_scanned:number, index_used:boolean, seq_scan_on_event_table:boolean}, required) — Live dashboard query must meet a latency/scan budget; the pasted EXPLAIN (ANALYZE, FORMAT JSON) plan is machine-parseable to prove index usage and bounded cost and to detect a Seq Scan on append-only event tables.
- `surface_binding_manifest` (object{widget_id:string, primary_rgb:string, series_palette:array<{series_key:string, token:string}>, render_states:object{loading:string, empty:string, error:string, partial:string, stale:string}, container_count:number, max_nesting_depth:number}, required) — Binds the query's categorical output to the actual rendered widget: every series color must reference an orgx-design token (tokens.md), render states must reference real component classes (components.md), and container/nesting counts make the 'every border is a failure of spacing' anti-pattern checkable.
- `surfaced_copy` (array<{element:string, text:string, kind:'label'|'value'|'empty'|'tooltip'|'error'}>, required) — User-facing strings are brand voice. Required so a deterministic lint can reject raw column names (agent_run_cnt), NaN/Infinity/null leakage ('Error: NaN%'), and uppercase-snake identifiers surfaced as labels — copy a senior reviewer rejects on sight.
- `claim_substantiation` (array<{surfaced_number:string, derivation:string, source_rows_ref:string, is_currency_or_performance:boolean, disclaimer:string}>, required) — Any number shown publicly (esp. $ or performance %) must trace to source rows. A check can require: every measure flagged is_currency_or_performance has a non-empty disclaimer AND its derivation references a column present in source_lineage.
- `definition_of_done_truth_set` (object{golden_dataset_sha:string, golden_rows:object[], expected_results:object[], tolerance:number, dispersion_stats:object{distinct_values:number, has_ties:boolean, has_nulls:boolean, stddev:number, skew:number}}, required) — A pinned ADVERSARIAL golden dataset + expected outputs makes correctness reproducible. dispersion_stats lets the gate REJECT a uniform/degenerate golden set where mean==median==p95 (the universal gaming path) and forces real spread, ties, and NULLs.
- `reference_evaluator` (string (path to executable JS/SQL that, given golden_rows, recomputes expected_results), required) — An independent executable that recomputes expected_results from golden_rows. The gate runs it and diffs against both expected_results and edge_case_fixtures.expected_output, so every 'expected' value is machine-verified rather than author-typed.

**Expected frontier pass rate:** Estimated 5–12% single-pass on the full gate. Reasoning: the arithmetic identities (CHK-RECOMPUTE-LINES/TOTALS, CHK-ROUNDING) are individually ~80–90% likely for a strong frontier model, but the gate has ~22 conjunctive checks and 13 are blockers, so pass probability is the product of many independent ~0.6–0.9 terms. The killers that frontier models reliably miss in one pass: (a) CHK-DISCLAIMERS-VERBATIM and CHK-CRM-NAME-BINDING require the SAME verbatim/normalized strings to appear in BOTH the HTML and an actually-parseable PDF — models routinely emit JSON data plus a stubbed or hash-inconsistent PDF (CHK-PDF-VALID); (b) CHK-NO-INTERNAL-LEAK + CHK-RENDER-DATA-FIDELITY are in tension — the model must render every line/total/disclaimer while scrubbing every cost/margin/approver value, and naive templating leaks margin or drops a clause; (c) CHK-COST-FROM-RECORD forces unitCost to equal a hashed snapshot AND every line to clear the margin floor, which a model optimizing the headline discount frequently violates; (d) CHK-AUTHORITY-TIER requires deriving the correct tier from a matrix AND producing a sha256 signature over the exact tuple — easy to get the hash inputs/order wrong; (e) CHK-MONEY-FORMAT/CHK-NO-OFFPALETTE-INLINE demand exact Intl formatting and a brand-constrained render with zero off-palette hex, which free-form HTML almost never satisfies on the first try; (f) every sha256/canonical-JSON field (rateCard, costRecord, crm snapshot, clause registry, approval signature) must be byte-exact, and canonicalization mistakes (key order, number formatting) break the hash. Because these failure modes are largely independent and several are ones current models systematically get wrong without iteration, the joint first-pass rate lands in the high single digits — comfortably below 100% and a meaningful production bar.

### Enriched artifact schema
- `quoteId` (string matching ^Q-[0-9]{4}-[0-9]{6}$, required) — Immutable CRM booking key; format is regex-checkable and must be unique vs supersededQuoteIds.
- `version` (integer >= 1, required) — Re-issued quotes must be ordered; booked version supersedes prior ones explicitly via supersedes[].
- `supersedes` (string[] of prior quoteId@version tokens) — Lets the gate verify version>1 explicitly references and voids prior versions (no silent dual-live offers).
- `currency` (string, ISO 4217 alpha-3 (member of bundled ISO4217 set), required) — Every monetary value is meaningless without it; membership in the ISO 4217 code set is a deterministic lookup.
- `locale` (BCP-47 tag (e.g. en-US, de-DE) used to drive Intl.NumberFormat, required) — Required so the gate can recompute the EXACT expected rendered money string (grouping, decimal mark, symbol placement) and diff it.
- `issueDate` (string, strict ISO 8601 YYYY-MM-DD, required) — Regex + Date.parse round-trip checkable; anchors the firm-offer window (UCC 2-205).
- `validThroughDate` (string, strict ISO 8601 YYYY-MM-DD, required) — Bounds the firm offer; must be strictly after issueDate — pure date arithmetic.
- `rateCard` (object {id, version(semver|date), sha256}, required) — Every unitListPrice must reconcile to this exact rate-card version; sha256 makes the source version tamper-evident and pinnable.
- `rateCardEntries` (array of {sku, listUnitPrice(int minor units), billingFrequency, ssp(int minor units)}, required) — The authoritative price source embedded in-artifact so unitListPrice/SSP claims are checkable by lookup, not assertion; must hash to rateCard.sha256.
- `lineItems` (array of {sku, description, quantity(int>0), billingFrequency('one_time'|'monthly'|'annual'), unitListPrice(int minor), listPriceSource, discountPct(number 0..1, <=4 decimals), discountReasonCode(enum), unitNetPrice(int minor), extendedPrice(int minor)}, required) — Real deliverable is multi-line; each line must reconcile to the rate card and carry a coded (not free-text) discount reason that maps to an authority tier.
- `subtotal` (integer (currency minor units), required) — Independently recomputable as sum of extendedPrice.
- `discountTotal` (integer (minor units), required) — Recomputable as sum(quantity*unitListPrice) - subtotal; ties out to per-line discounts.
- `taxLines` (array of {jurisdiction, rate(0..1), taxableBase(int minor), taxAmount(int minor)}, required) — Tax must be independently recomputable per jurisdiction with banker's rounding; ASC 606 distinct amounts.
- `total` (integer (minor units), required) — Recomputable as subtotal + sum(taxAmount); the single number the customer owes.
- `costRecord` (object {systemOfRecordId, asOfDate(ISO8601), sha256, entries:[{sku, unitCost(int minor)}]}, required) — Closes the self-attested-cost hole: unitCost used by marginFloorCheck must come from this hashed costing snapshot, not be invented; sha256 pins it to a system of record.
- `marginFloorCheck` (array per-line {sku, unitCost(int minor), grossMarginPct(number), floorPct(number), pass(boolean)}, required) — unitCost must equal costRecord.entries[sku].unitCost (lookup, not assertion); margin recomputed from net price; a discount within authority can still violate the floor.
- `discountAuthority` (object {effectiveDiscountPct, requiredTier('rep'|'manager'|'vp'|'finance'), authorityMatrix:[{maxPct, tier}], approverId, approverName, approverTier, approvalTimestamp(ISO8601), approvalEvidenceRef(uri/id), approvalSignatureSha256}, required) — SOX delegation-of-authority: effective discount maps to a required tier via the embedded matrix; approverTier must be >= required; approval must carry a hash-verifiable signature/evidence ref, not the word 'authorized'.
- `clauseRegistry` (object {version, clauses:[{key, text, sha256}]}, required) — Canonical disclaimer/clause library embedded so 'verbatim' becomes an exact sha256(normalize(text)) match instead of keyword/semantic judgment.
- `requiredDisclaimerKeys` (string[] (must include 'tax_exclusivity','validity','non_binding_until_countersigned','auto_renewal'), required) — Names which canonical clauses MUST be present; gate verifies each key exists in clauseRegistry and its verbatim text appears in the rendered doc.
- `priceClaims` (array of {claimText, claimType('pct_off'|'amount_off'|'was_now'|'list'), referencePrice(int minor), referenceSource('rateCard'|'priorQuote'), referenceProofRef, referenceEffectiveFrom(ISO8601), referenceEffectiveTo(ISO8601|null)}) — FTC 16 CFR 233: any savings/was-now claim must reference a substantiated price; gate checks referencePrice equals the cited rateCard SSP / prior quote value and that the reference window is a real, bounded period (substantiation reduced to a pin + date-window check).
- `assumptions` (string[], required) — Term length, seat count, what is NOT included — required non-empty so the quote is a real scoped offer, not a bare number.
- `exclusions` (string[], required) — Explicit out-of-scope list; required non-empty to bound the offer.
- `crmBinding` (object {opportunityId, accountId, contactId, ownerId, stage, snapshot:{accountName, contactName, contactEmail, recordsSha256}}, required) — 'Booked in CRM' = references real records; the snapshot + recordsSha256 lets the gate verify the customer name/email rendered in the doc matches the bound CRM record exactly (kills mail-merge/'Dear [CUSTOMER_NAME]' failures).
- `brandRegistry` (object {brandId, logoSha256, approvedFontFamilies:string[], approvedColorTokens:[{name, hex}], primaryColorToken}, required) — Makes brand identity checkable: rendered doc must embed the logo with this sha256, use only approved fonts, and apply the primary brand color — no off-palette/system-font output.
- `renderArtifacts` (object {html(string), pdfBase64(string), pdfBytes(int), logoSha256(string), fontFamiliesUsed:string[], colorTokensUsed:[{name,hex}], customerFacingFieldWhitelist:string[]}, required) — The document the customer actually receives. Must contain the line data/totals/disclaimers, use brand tokens, and NOT contain internal cost/margin/approver fields. pdfBytes + parse check kills empty/lorem PDFs.
- `computationTrace` (object {roundingPolicy('half_even'), minorUnitDigits(int), formulaPerLine:[{sku, formula, inputs, result}]}, required) — Every number reproducible from named inputs + formula + a SINGLE declared rounding policy (banker's rounding) at a declared minor-unit precision — no unexplained values, no compounding drift.

## Product prioritized backlog (RICE) — a quarter-committed, leadership-socialized roadmap rendered as a UI/doc artifact. The deliverable is a ranked list of candidate initiatives scored on RICE (Score =

**Ship gate:** SHIP only if: (1) ALL blocker-severity checks pass with their stated thresholds — SCHEMA-WELLFORMED, CORR-RICE-FORMULA, CORR-IMPACT-SCALE, CORR-CONFIDENCE-TIER, CORR-EFFORT-BREAKDOWN, RANK-ORDER, DEP-RESOLVABLE, SEQ-RANK-RESPECTS-DEPS, PROV-SOURCES, PROV-REACH-DERIVATION, CAP-COMMITTED-FITS, RENDER-VARIANTS-DISTINCT, RENDER-TOKEN-CONFORMANCE, RENDER-TEXT-CONTRAST, RENDER-FOCUS-VISIBLE (zero tolerance, no waivers); AND (2) at most ONE major-severity check fails and it is documented with a written waiver and a fix-by date in changelog; AND (3) minor-severity failures are logged but non-blocking. Any single blocker failure, or two-or-more major failures, blocks the quarter commitment. All checks must execute in plain JS by parsing the JSON artifact plus the two rendered HTML variants against the token allowlist resolved from rendering.tokens_ref — no LLM judge participates in any gate decision.

**Expected frontier pass rate:** Estimated 12–22% single-pass on the full gate. Reasoning: a frontier model reliably nails the deterministic-arithmetic and structural cluster (SCHEMA, CORR-RICE-FORMULA, CORR-IMPACT-SCALE, CORR-CONFIDENCE-TIER, CORR-EFFORT-BREAKDOWN, RANK-ORDER, DEDUP) — roughly 90%+ each — because those are mechanical. The pass rate collapses on the conjunction of harder gates: (a) PROV-REACH-DERIVATION requires every reach number to be numerically reconstructable from source-tied factors with the unit named (models routinely emit bare reach values) — ~55%; (b) the rendering cluster requires producing TWO distinct HTML files that parse, declare correct theme backgrounds, use ONLY the resolved OrgX token palette with zero hardcoded hex, pass WCAG-AA text contrast in both themes, and give every interactive element a >=3:1 focus indicator in both themes — single-pass models almost always leak at least one off-palette hex or omit focus-visible rules — RENDER-TOKEN-CONFORMANCE ~45%, RENDER-FOCUS-VISIBLE ~40%, RENDER-TEXT-CONTRAST ~60%, jointly ~15–20%; (c) CAP-COMMITTED-FITS + CAP-CUTLINE-NONTRIVIAL + CAP-CAPACITY-SOURCED force a committed set that both fits sourced capacity with a 5–40% buffer AND consumes >=50% of it without exceeding it — a tight band models often miss — ~55%; (d) SEQ-RANK-RESPECTS-DEPS + CONSIST-DEP-DECLARED catch the common shortcut of empty dependency arrays alongside titles that reference other items. Because the gate is a conjunction of ~15 blockers, the joint single-pass probability is the product of the weak links (~0.55 × ~0.18 × ~0.55 × ~0.85 ≈ 0.046 for a naive attempt, rising to ~0.12–0.22 once a competent model deliberately addresses rendering and provenance), landing well below 100% and concentrated almost entirely in the brand/accessibility rendering surface and provenance reconstruction.

### Enriched artifact schema
- `items[]` (array<BacklogItem> (length>=1), required) — Backlog must be a structured machine-checkable list, not prose, so every per-item check runs deterministically.
- `items[].id` (string regex ^[A-Z]{2,5}-[0-9]{1,4}$ (unique), required) — Stable identifier for traceability, dependency references, and dedup; free-text titles cannot be diffed or linked.
- `items[].title` (string (3..10 words, ^[A-Z], no trailing period, no emoji, not in banned-AI-filler set), required) — Human-readable roadmap line; bounded + banned-word so the socialized roadmap reads consistently and resists AI-ese hallucination.
- `items[].reach` (object {value:number>0, unit:enum[users/quarter,accounts/quarter,sessions/quarter], window:const '1 quarter', source_ref:string, derivation:string}, required) — Reach must be a real sourced count over a fixed window per standard RICE; source_ref ties it to a query/dashboard, derivation shows the arithmetic so it cannot be a bare fabricated number.
- `items[].impact` (number enum {0.25,0.5,1,2,3}, required) — Intercom RICE constrains Impact to the discrete massive=3/high=2/medium=1/low=0.5/minimal=0.25 scale; free-form impact breaks comparability.
- `items[].confidence` (number enum {0.5,0.8,1.0}, plus confidence_basis:enum[has_data,has_estimate,has_anecdote], required) — Confidence is the de-risking discount and must map to evidence tier (100%/80%/50% per Intercom) so a model cannot assign 100% to an unvalidated guess; basis must be consistent with the value.
- `items[].effort` (object {person_months:number>0, breakdown:{design:number>=0,eng:number>=0,qa:number>=0}, estimator:string}, required) — Effort must be person-months with a role breakdown that sums to the total so capacity is reconcilable and zero/near-zero-effort gaming is impossible.
- `items[].effort_unit_assertion` (const 'person-months', required) — Forces one consistent effort unit across all items so scores are comparable; mixing weeks and months silently distorts ranks.
- `items[].score` (number (computed), required) — RICE score; must equal (reach.value*impact*confidence)/effort.person_months within tolerance so arithmetic is independently re-derivable.
- `items[].rank` (integer 1..N (unique, contiguous), required) — Committed ordinal position; checked to strictly follow score and to fit the capacity cut-line.
- `items[].sources[]` (array<{ref:string(url|query_id|doc#section), claim:string, retrieved_at:ISO8601}> (length>=1), required) — Every estimate must trace to a named source with a retrieval timestamp; provenance is what separates a real backlog from theater.
- `items[].assumptions[]` (array<string> (>=1 when confidence<1.0), required) — Explicit falsifiable assumptions; an item with confidence<1.0 but zero assumptions is internally inconsistent.
- `items[].dependencies[]` (array<item.id> (referential, acyclic), required) — Sequencing correctness: a higher-ranked item cannot depend on a lower-ranked or out-of-quarter item; refs must resolve and form no cycle.
- `capacity` (object {team_capacity_person_months:number>0, committed_person_months:number, cutline_rank:integer 1..N, buffer_pct:number in [0,100]}, required) — A quarter commitment is meaningless unless the committed set (rank<=cutline) fits real capacity with a stated buffer; the cut-line makes it falsifiable.
- `scope_universe` (object {candidates_considered:integer>=items.length, candidates_scored:integer, intake_source:string}, required) — Completeness vs the real deliverable; candidates_scored must equal items.length and considered>=scored proves the backlog is not a toy subset.
- `rendering` (object {tokens_ref:string, light_artifact_ref:string, dark_artifact_ref:string}, required) — The socialized roadmap is a real UI artifact: tokens_ref resolves to the OrgX token file; light_artifact_ref/dark_artifact_ref each resolve to a distinct non-empty rendered HTML file that is actually parsed and checked (not self-asserted booleans).
- `disclaimers` (object {estimate_disclaimer:string (8..40 words), forecast_basis:string (>=5 words), last_recompute_at:ISO8601}, required) — Reach/impact are forward-looking projections shown to leadership; a bounded on-voice estimate disclaimer + forecast basis + recompute timestamp prevent the roadmap being read as a guaranteed outcome (FTC-style substantiation).
- `changelog[]` (array<{at:ISO8601, item_id:string, field:string, from:any, to:any, rationale:string(>=3 words)}>, required) — Any change to a score or rank must be auditable so leadership can trust the committed order did not silently drift; rationale must be non-trivial.

- **CORR-RICE-FORMULA** [blocker, correctness] — Every item's score equals (reach.value × impact × confidence) / effort.person_months.
  - compute: for each item: expected=(reach.value*impact*confidence)/effort.person_months; assert abs(item.score-expected) <= max(0.5, 0.005*expected). FAIL if any item exceeds tolerance.
  - threshold: 0 items out of tolerance (>0.5% relative or >0.5 absolute)
  - gameability: Passable with fabricated inputs — only proves arithmetic. Provenance (PROV-SOURCES) and scale (CORR-IMPACT-SCALE/CORR-CONFIDENCE-TIER) checks are what make the inputs real.
- **CORR-IMPACT-SCALE** [blocker, standards] — Impact is one of the discrete Intercom RICE values {0.25,0.5,1,2,3}.

  - gameability: Setting everything to 1.0/has_data is blocked by PROV-CONFIDENCE-EVIDENCE which requires a data-tier source for confidence==1.0.
- **CORR-EFFORT-BREAKDOWN** [blocker, correctness] — effort.person_months > 0 and equals design+eng+qa within tolerance; effort_unit_assertion is the literal 'person-months'.

  - gameability: Tiny effort (e.g. 0.01) inflates score; CAP-EFFORT-FLOOR enforces a realistic per-item floor and capacity reconciliation catches unrealistic totals.
- **CAP-EFFORT-FLOOR** [major, robustness] — No item uses an implausibly small effort to game its score; each effort.person_months >= 0.25 (≈ 1 person-week).
  - compute: for each item assert effort.person_months>=0.25. FAIL count of items below floor.
  - threshold: 0 items below 0.25 person-months
  - gameability: A model could set exactly 0.25 across the board; combined with CAP-COMMITTED-FITS the sum still must fit real capacity, bounding total understatement.
- **RANK-ORDER** [blocker, correctness] — Ranks are a contiguous unique 1..N permutation and strictly follow score descending (ties broken by lower effort then id).

  - threshold: 0 rank mismatches; ranks are exact 1..N permutation
  - gameability: Deterministic given scores; the value comes only when scores themselves are real (gated by provenance/scale checks).

  - gameability: Empty dependencies arrays trivially pass; SEQ-RANK-RESPECTS-DEPS only bites when dependencies exist, and CONSIST-DEP-DECLARED flags suspiciously empty graphs vs prose.

### Enriched artifact schema
- `claims[]` (array of objects, one per published atomic assertion (every number, %, $, ratio, superlative, comparative, capability/outcome phrase, third-party statistic, pricing term) across ALL listed artifacts, required) — Acceptance is per-claim, not per-document. A doc-level pass lets one fabricated number ride 50 true ones. Each assertion must be individually addressable and gated.
- `claims[].id` (string (stable, e.g. c1.tile.value_created), required) — Checks, the coverage binding, and the legal signoff all reference claims by id; without a stable id the extraction-coverage check (CLM-02) cannot prove the list is complete.
- `claims[].text` (string (verbatim as rendered to the public, incl. units/qualifiers/casing), required) — Substantiation must match what the reader sees, not a paraphrase. '$468' and 'over $450' are different claims with different proof burdens; '71/100' and '71%' differ.
- `claims[].artifactPath / claims[].surface` (string filepath + enum {html, png, markdown}, required) — The same claim renders differently across the HTML (DOM, checkable) and the flattened PNG (no DOM). The published surface of record for ads is the PNG; checks must know which surface each claim lives on.
- `claims[].renderedText` (string — OCR/extracted text of the exact PNG region this claim occupies (with bbox {x,y,w,h}), required) — PNGs have no DOM. 'disclaimer placed in footer' is unfalsifiable against a raster unless the text is OCR-extracted from the actual pixels. This field is what CLM-02/CLM-12/CLM-13 run against.
- `claims[].claimType` (enum: quantitative_metric | comparative | superlative | capability | outcome_promise | testimonial | third_party_stat | pricing, required) — FTC substantiation burden differs by type. Comparative/superlative/outcome claims require typicality or head-to-head evidence, not one datapoint.
- `claims[].source` (object {type: sql|http|invoice|third_party_url|screenshot, locator, executableQuery, retrievedAt(ISO8601), value, unit, snapshotHash}, required) — 'Trace to a fact sheet' is gameable — anyone hand-types a fact sheet. Source must be a RE-EXECUTABLE locator: for sql/http the executableQuery is runnable code (NOT prose), with a snapshotHash of the dataset it ran against so CLM-01 re-runs deterministically against a frozen snapshot.
- `claims[].computationCode` (string — RUNNABLE SQL/JS (filter, window, dedup, rounding) — NOT an English description, required) — Reviewers reading a prose formula ('value created, 30d, dedup by initiative') produce different queries. A deterministic re-derivation requires the aggregation to be executable code. Given the repo's 6,771-duplicate-initiative and $0-cost-bug history, the dedup rule must be in code, not prose, or the SUM is silently wrong.
- `claims[].asOfDate / windowStart / windowEnd` (ISO8601, required) — '30d'/'this month' are time-relative; a number true in June is false in July. Static PNGs need a frozen as-of date and the covered window or the claim silently rots.
- `claims[].relativeTimeWords` (array<string> — relative time tokens detected in renderedText (e.g. 'this month','30d','today'), required) — Drives CLM-07: any creative whose visible copy uses a relative time word MUST visibly render an as-of date in the same raster, else the asset is false to a July reader while green in the gate.
- `claims[].roundingDisclosure` (object {mode: exact|rounded, direction, precision, rawValue}, required) — Headline literally says 'nothing rounded up'. rawValue lets CLM-06 verify the self-declared mode: if mode=exact but rawValue != published, the claim is mislabeled; the cross-claim headline-consistency check (CLM-06) then fires.
- `claims[].typicality` (object {n:int, isTypicalResult:boolean, requiredDisclaimer:string, disclaimerRendered:boolean}, required) — FTC requires results claims to reflect typical experience or carry a clear disclaimer. A '$468' self-dogfood number presented as representative without n and a 'our own usage, results vary' disclaimer is deceptive; disclaimerRendered must be provable in the raster (CLM-12).
- `colorTokens[]` (array of {selector, role, fgHex, fgAlpha, bgHex, computedContrast} for EVERY distinct text/number element incl. sub-tokens (.sub .url .kick .rl .cap .meta), required) — The old 4-token enum {fg,bg,lime,teal} never modeled the low-alpha secondary text (.url at .32 ~= 2.0:1, fails AA) where the as-of/disclaimer/url actually live. Per-element token+computed alpha-composited contrast is the only way that text gets checked (CLM-08).
- `stateVariants` (object {dark (on #02040a), lightUnfurl (OG/link preview on light bg), focusState (interactive/linked), reducedMotion (still for any GIF/video frame)}, required) — Claims appear on dark canvas AND in OG/unfurls on light backgrounds; each surface needs a declared contrast-valid variant, and GIF/video frames need a reduced-motion still (CLM-08, CLM-11).
- `altText` (string per image conveying the claim AND its as-of/typicality qualifier; MUST contain every number string in that creative, required) — A proof card is a data graphic; the screen-reader version must carry the same numbers and the same qualifier, else the accessible version makes an UNQUALIFIED (more deceptive) claim. Checked structurally: contains all numeric tokens + the as-of date (CLM-09).
- `legalReviewSignoff` (object {reviewer, role, date(ISO8601), ftcChecklistVersion, perClaimSignoff: map<claimId, {approved:boolean, disclaimerLocatorBbox}>}, required) — Ships to paid ads (FTC + platform policy). - `brandLexicon` (object {bannedTermsFound:[], requiredCasing, subjectLines:[{text,wordCount,isLowercase,bannedHit}], taglines:[{text,charCount}]}, required) — Kill-list + casing + <=6-word lowercase subject rule + tagline <=60 are lexically machine-checkable. **Expected frontier pass rate:** Estimated single-pass frontier pass rate: 8-15%. Reasoning: A frontier model reliably produces a polished, on-brand-looking bar chart with plausible spec/data JSON, good contrast, and coherent claims — so the 'soft' checks (BRAND-09, A11Y-14, CLAIM-06) pass most of the time. The pass rate collapses on the checks that require an EXTERNAL ground truth and exact structural agreement: (a) SRC-17/REPRO-18 — provenance locators must resolve in a real gate-supplied source file to the exact claimed values and the builder must regenerate byte-identical hashes; a model generating the artifact in one shot almost never has truly resolvable locators + deterministic hashing against an unseen source, so most single-pass attempts fail here outright. (b) DATA-03 uniform exclusion (excluded set must EQUAL the predicate-matched set over all source rows) catches the near-universal habit of dropping inconvenient rows ad hoc. (c) BRAND-11 token-actually-consumed and COMP-12 geometry-re-parse catch spec/render drift, which is extremely common since models author the JSON and the SVG semi-independently. (d) VAR-13 demands three byte/structure-equivalent variants — models routinely emit one good variant and stub the others. (e) FACE-16 on-face furniture + logo clear-space is frequently omitted. Each blocker is independent and failure-prone; with ~17 blockers where several have <60% independent single-pass success and 2-3 (SRC-17/REPRO-18/VAR-13) have <40%, the joint pass probability lands in the low double digits at best — well below 100%, as intended.

### Enriched artifact schema
- `chart.spec.encoding` (object {xField:string, yField:string, aggregate:'sum'|'mean'|'count'|'none', scale:{type:'linear'|'log', zero:boolean, domain:[number,number], breakMarker:boolean}, sort:{by:string, order:'asc'|'desc'|'declaredCategoryOrder'}}, required) — Machine-readable encoding so axis-baseline, scale type, sort, domain, and aggregation are checked deterministically against the rendered SVG instead of inferred from pixels. sort makes ordering an explicit contract rather than taste.
- `data.rows[]` (array<{rowId:string, category:string, value:number, valueRaw:number|null, unit:string, period_iso8601:string, displayDecimals:integer}>, required) — The exact plotted rows in machine-readable form. valueRaw vs value exposes any transform; null marks genuine missing data so it cannot be silently dropped or zero-filled; displayDecimals fixes the precision tolerance for fidelity reconciliation.
- `data.provenance[]` (array<{rowId:string, sourceSystem:string, sourceLocator:string, extractedAt:ISO8601, sourceValue:number, transformChain:string[]}>, required) — Every plotted number traces to a named source location (query string, spreadsheet cell ref, API endpoint+path). sourceLocator must be resolvable against the gate-supplied source file so provenance cannot be fabricated. transformChain lists transformIds applied in order, making value reconstruction deterministic.
- `data.transforms[]` (array<{transformId:string, op:'unit_convert'|'dedupe'|'impute'|'outlier_clip'|'currency_normalize'|'reaggregate'|'fx_rate', params:object, rowsAffected:string[], inverseDeterministic:boolean}>, required) — The messy-data cleanup must be declared, not hidden, and each transform must be a pure deterministic function of (sourceValue, params) so the gate can replay it. Lets the reviewer verify imputation/clipping/dedup did not fabricate or distort values.
- `data.exclusions[]` (array<{rowId:string, reason:string, ruleId:string}>, required) — Any source row NOT plotted must be listed with a rule-based reason whose ruleId resolves to data.exclusionRules. Deterministic anti-cherry-pick control: exclusions must follow a stated rule applied uniformly, not taste.
- `data.exclusionRules[]` (array<{ruleId:string, predicate:string}>, required) — A whitelisted, machine-evaluable predicate (e.g. 'sourceValue==null', 'period<2023-01-01') for every exclusion. The gate evaluates each predicate against ALL source rows and asserts the excluded set equals exactly the rows matching some rule — closing selective-exclusion (cherry-pick) by computation.
- `data.completeness` (object {expectedCategories:string[], expectedPeriods:string[], renderedCategories:string[], renderedPeriods:string[], populationSource:string}, required) — Lets a script confirm the chart covers the FULL deliverable population (derived from populationSource in the real source file), not a flattering subset. expected* must be derived from source, not self-asserted.
- `claims[]` (array<{claimId:string, text:string, anchorRowIds:string[], computation:string, expectedValue:number, tolerance:number, surface:'title'|'callout'|'annotation'|'subtitle'}>, required) — Every textual/numeric claim (e.g. 'Q4 up 23%') binds to source rows + an explicit machine-evaluable formula (whitelisted arithmetic DSL) so the gate recomputes expectedValue and matches the number lexically extracted from the rendered text. surface lets the gate locate the exact text node.
- `disclaimers` (object {forwardLooking:string|null, nonGaapReconciliation:{ref:string, resolves:boolean, reconciledLineItems:[{gaap:string, nonGaap:string, deltaExplained:number}]}|null, asOfDate:ISO8601, dataVintage:ISO8601, units:string, currency:string}, required) — Investor materials legally require forward-looking safe-harbor language, non-GAAP reconciliation (Reg G), as-of/units/currency. The reconciliation is now structured so the gate verifies the referenced doc resolves AND the line-item deltas sum/balance — presence alone no longer passes.
- `a11y` (object {svgRole:string, title:string, desc:string, dataTableId:string, seriesNames:string[], focusOrder:string[]}, required) — WAI-ARIA Graphics Module requires a programmatic name/desc and an equivalent data-table fallback. Adequacy is enforced structurally: the data table must contain every category+value, and desc must mention chart type + series count + extrema (checked lexically), not just be non-empty.
- `render.tokens` (object {colorTokens:string[], fontTokens:string[], spacingTokens:string[]}, required) — Maps every visual color/font/spacing to a named design-system token. Used together with render.elements so the gate verifies the rendered SVG actually CONSUMES these tokens at the right roles, not merely that the strings are DS members.
- `render.elements[]` (array<{elementId:string, role:'dataSeries'|'emphasisBar'|'axis'|'gridline'|'title'|'subtitle'|'legend'|'callout'|'sourceLine'|'asOfStamp'|'logo', tokenId:string, svgSelector:string}>, required) — Binds each rendered SVG node to its semantic role and the DS token it uses, so token-ROLE conformance (e.g. error/destructive token never on a neutral data bar; emphasis token used on exactly one bar) is a deterministic membership+count check against the actual render.
- `render.variants` (object {light:{svgRef:string, dataHash:string}, dark:{svgRef:string, dataHash:string}, print_grayscale:{svgRef:string, dataHash:string}}, required) — Dashboard has dark mode and the deck prints. Each variant must independently pass contrast/CVD AND carry a dataHash so cross-variant content equivalence (same bars, order, claims, heights) is verifiable — preventing a pristine light + degraded dark mismatch.
- `render.geometry` (object {plotArea:{x:number,y:number,width:number,height:number}, bars:[{rowId:string, x:number, y:number, width:number, height:number}], barGap:number, aspectRatio:number, plotMargins:{top:number,right:number,bottom:number,left:number}, gridUnit:number, tickLabelCount:integer}, required) — Pre-extracted SVG geometry so composition/hierarchy checks (gap-to-width ratio, margins on the spacing grid, aspect ratio in allowed set, tick density) are pure numeric range checks. The gate independently re-parses the SVG and asserts geometry matches this declared block.
- `qa.contrastReport` (array<{pair:string, fg:string, bg:string, ratio:number, wcagLevel:'AA'|'AAA'|'fail', variant:'light'|'dark'|'print_grayscale', context:'text'|'nonTextGraphic'}>, required) — Pre-computed contrast ratios for every foreground/background and adjacent-bar pair, per variant, so the gate re-verifies (recomputes from the actual rendered colors) rather than trusting the report.
- `qa.colorBlindSim` (object {deuteranopia:{minAdjacentDeltaE:number}, protanopia:{minAdjacentDeltaE:number}, tritanopia:{minAdjacentDeltaE:number}, nonColorEncodingPresent:boolean}, required) — Bars distinguished by hue alone fail CVD users; simulated CIEDE2000 separation is measured per CVD type with a numeric pass threshold, and a redundant non-color encoding (pattern/label/direct value) must be present so distinguishability never depends on hue alone.
- `checksum` (object {dataHash:sha256, specHash:sha256, sourceFileHash:sha256, builderVersion:string, generatedAt:ISO8601}, required) — Reproducibility: re-running the pipeline on the same source must regenerate identical hashes, and sourceFileHash pins the EXACT source artifact the gate re-runs against, defeating self-consistent fabrication.

### Checks (20)
- **DATA-01-fidelity** [blocker, correctness] — Every rendered bar height maps to its declared data value within rounding tolerance, and that value reconciles to the source value after the declared transform chain.
  - compute: For each render.geometry.bars[i]: measuredFrac = bar.height / plotArea.height; expectedFrac = (value_i - scale.domain[0]) / (scale.domain[1]-scale.domain[0]); assert abs(measuredFrac-expectedFrac)*plotArea.height <= 1. Then replay: v = provenance[rowId].sourceValue; for tId in transformChain: v = applyTransform(transforms[tId].op, v, params); assert abs(v - value_i) <= 0.5*10^(-displayDecimals).
  - threshold: 100% of bars within 1px AND value reconciles to replayed source value to displayed precision
  - gameability: A model can hardcode bar heights to match a values array it also authored, so heights agree with themselves. Binding value->replayed sourceValue closes this ONLY because provenance.sourceLocator is resolved against the gate-supplied real source file in SRC-13/REPRO-12; without that re-run a model fabricates plausible provenance and passes.
- **DATA-02-baseline-zero** [blocker, correctness] — Linear value axis baseline starts at zero (no truncated y-axis); a nonzero baseline is allowed only when explicitly marked AND disclosed AND the rendered axis agrees.
  - compute: Parse the lowest y-axis tick label text from the SVG. assert (encoding.scale.type=='linear' AND encoding.scale.zero==true AND lowestTickLabelNumeric==0) OR (encoding.scale.zero==false AND encoding.scale.breakMarker==true AND svg contains an element with role/class 'axis-break' AND disclaimers text references axis truncation). Additionally assert the rendered bottom gridline maps to domain[0] within 1px so spec and render cannot drift.
  - threshold: zero-baseline confirmed in BOTH spec and rendered axis label/gridline, OR fully-marked+disclosed+rendered-break exception; else fail
  - gameability: Model sets scale.zero=true in JSON while the rendered axis starts nonzero (spec/render drift). Re-parsing the lowest tick label AND the bottom gridline pixel position from the actual SVG closes the drift; the editorial 'is the truncation justified' judgment has been REMOVED (non-deterministic) and replaced by the strict requirement that any nonzero baseline carries a visible break marker + disclosure.
- **DATA-03-exclusion-rule-uniform** [blocker, correctness] — Every excluded source row matches a declared exclusion rule, and EVERY source row matching a rule is excluded (no selective/cherry-pick exclusion).
  - compute: Load all source rows from the gate-supplied source file. For each rule in exclusionRules, evaluate its whitelisted predicate over every source row -> matchedSet = union of all rule matches. excludedSet = {e.rowId for e in exclusions}. plottedSet = {r.rowId for r in data.rows}. assert excludedSet == matchedSet (set equality). assert plottedSet ∪ excludedSet == allSourceRowIds AND plottedSet ∩ excludedSet == ∅. assert every exclusion.ruleId ∈ exclusionRules.
  - threshold: excludedSet exactly equals predicate-matched set; plotted+excluded partitions the full source population with no gaps/overlaps
  - gameability: A model excludes inconvenient rows with a bespoke 'reason' string. Requiring exclusion via a machine-evaluable predicate applied UNIFORMLY to all source rows means it cannot drop one weak quarter while keeping a comparable one — the predicate would catch both. Predicates are restricted to a whitelisted comparison DSL so no arbitrary code/judge is involved.
- **DATA-04-completeness-population** [blocker, completeness] — The chart covers the full expected category/period population derived from the source, not a flattering subset.
  - compute: Derive expectedCategories/expectedPeriods by evaluating completeness.populationSource against the source file (e.g. distinct categories after exclusionRules). assert set(data.completeness.expectedCategories) == derivedCategories AND set(renderedCategories) == set(bars.category). assert renderedCategories ⊇ (expectedCategories minus rows excluded by rule). Any expected category absent from render that is NOT covered by an exclusion rule -> fail.
  - threshold: rendered category/period set equals expected-minus-ruled-exclusions exactly
  - gameability: Self-asserted expectedCategories could be trimmed to match a cherry-picked render. Deriving expected* from the source file via populationSource (not trusting the artifact's own array) blocks this.
- **CLAIM-05-recompute** [blocker, correctness] — Every numeric claim in title/subtitle/callout/annotation recomputes from its anchor rows and matches the number shown in the rendered text.
  - compute: For each claims[c]: evaluate computation (whitelisted arithmetic DSL over anchorRowIds' values) -> computed. assert abs(computed - c.expectedValue) <= c.tolerance. Locate the text node for c.surface/claimId in the SVG; extract all numerics via regex; assert c.expectedValue (formatted to displayDecimals) appears in that node's normalized text. assert every numeric token present in title/subtitle/callout/annotation text nodes is covered by some claim (no un-anchored numbers).
  - threshold: 100% of claims recompute within tolerance AND appear in the matching rendered text node; zero un-anchored numerics in annotation text
  - gameability: Editorial 'is the framing hyperbolic / honest' judgment was REMOVED as non-deterministic. What remains is purely arithmetic + lexical: every shown number must be recomputable from anchored source rows, and no number may appear without an anchor — which structurally blocks invented stats even if it cannot judge tone.
- **CLAIM-06-number-format-consistency** [major, consistency] — All numeric labels (axis ticks, bar value labels, callouts) use a single consistent format and unit string.
  - compute: Extract every numeric label text node from the SVG. Classify each against a regex format signature (thousands separator, decimal places, scale suffix like K/M, currency symbol position). assert count(distinct formatSignatures among value labels) == 1. assert every unit/currency mention in text nodes ∈ {disclaimers.units, disclaimers.currency} and the canonical unit string is rendered identically everywhere (string equality after normalization).
  - threshold: exactly one numeric format signature across value labels; unit/currency strings byte-identical wherever they appear
  - gameability: Mixed '1,200' vs '1.2K' vs '$1200.00' and three different unit spellings all previously passed. This is a deterministic format-signature equality check; it does not judge which format is 'nicer', only that drift is zero.
- **DISC-07-reg-g-reconciliation** [blocker, legal] — Investor-facing chart carries forward-looking safe-harbor text, and any non-GAAP figure has a resolvable Reg G reconciliation that actually balances.
  - compute: assert disclaimers.asOfDate, dataVintage, units, currency are non-empty and valid ISO/format. If any claim or bar is tagged non-GAAP (transform op ∈ {currency_normalize, reaggregate, outlier_clip} influencing a headline value), then disclaimers.nonGaapReconciliation must be present, .resolves==true (gate dereferences ref and confirms target exists), and for each reconciledLineItem assert abs((gaapValue + deltaExplained) - nonGaapValue) <= tolerance using values pulled from the reconciliation target. assert forwardLooking is non-empty when any claim.surface contains a future-period token (regex for years/'guidance'/'projected').
  - threshold: as-of/units/currency present; Reg G reconciliation resolves AND line-item deltas balance; safe-harbor present when forward-looking language detected
  - gameability: Boilerplate text + a dangling ref previously passed on presence. Now the gate dereferences the ref (must resolve) and arithmetically checks the GAAP->non-GAAP bridge balances, so an unsubstantiated reference fails.
- **A11Y-08-equivalent-table** [blocker, accessibility] — A programmatic name/desc and an equivalent data-table fallback exist and actually contain the full plotted data and chart semantics.
  - compute: assert svg[role]=='img' or 'graphics-document'; assert aria-labelledby/aria-describedby resolve to title/desc nodes. Locate dataTableId element; parse rows; assert it contains one row per data.rows entry with matching category and value (to displayDecimals). assert desc text contains: the literal chart type ('bar'), the series count, and the min and max category labels (lexical presence check). assert focusOrder lists every interactive/series element id exactly once.
  - threshold: data table is a 1:1 match of plotted rows; desc mentions type+count+extrema; focusOrder complete and duplicate-free
  - gameability: Schema-presence ('desc is a non-empty string') previously let 'chart of data' pass. Adequacy is now structural: the table must reproduce every plotted value and the desc must contain specific computable tokens. This does not judge prose quality, only verifiable equivalence.
- **BRAND-09-token-membership** [major, brand] — Every color/font/spacing token used is a member of the design-system namespace.
  - compute: Load DS token registry (gate-supplied). assert every id in render.tokens.colorTokens ⊆ DS.color, fontTokens ⊆ DS.font, spacingTokens ⊆ DS.spacing. For every render.elements[e].tokenId, assert it ∈ render.tokens of the matching category.
  - threshold: 100% set-membership; zero unknown tokens
  - gameability: Necessary but not sufficient — pairs with BRAND-10 (role binding) and BRAND-11 (consumption) so passing membership alone no longer implies on-brand. Membership itself is fully deterministic.
- **BRAND-10-token-role-binding** [blocker, brand] — Each token is applied only to a semantically permitted role; emphasis/destructive tokens are not misused on neutral data bars.
  - compute: For each render.elements[e]: assert e.tokenId ∈ DS.allowedTokensForRole(e.role). assert distinct dataSeries colors ⊆ DS.dataPalette AND count(distinct dataSeries colors) == count(series). assert no token tagged DS.role ∈ {error,destructive,warning} appears on any element with role ∈ {dataSeries, gridline, axis}. assert exactly one element has role 'emphasisBar' and it uses a token ∈ DS.accentPalette (or zero emphasis bars if encoding declares none).
  - threshold: zero role violations; emphasis token used on at most the single declared emphasis bar
  - gameability: A red 'destructive' token on a neutral revenue bar is a valid set member but a brand failure; the editorial 'tasteful' judgment is REMOVED and replaced by an enumerated role->allowed-token mapping from the DS, which is pure set membership.
- **BRAND-11-token-actually-consumed** [blocker, brand] — The rendered SVG actually consumes the declared tokens at the declared selectors (no declared-but-unused or undeclared-but-rendered styling).
  - compute: For each render.elements[e]: resolve e.svgSelector in the actual SVG; read its computed fill/stroke/font-family/spacing; resolve the literal value of e.tokenId from DS; assert rendered literal == DS token literal (hex/font/px equality within rounding). assert every fill/stroke/font on a visible node is traceable to some render.elements entry (no orphan inline styles).
  - threshold: every declared element's rendered style equals its DS token literal; zero orphan/undeclared visible styles
  - gameability: Closes the 'declares correct tokens in JSON but renders something else' gap. Pure literal comparison between resolved DS token value and the value actually present in the SVG node.
- **COMP-12-layout-geometry** [blocker, brand] — Composition geometry (bar gap/width ratio, margins, aspect ratio, tick density, baseline grid snap) falls inside design-system-declared ranges.
  - compute: Re-parse SVG geometry independently and assert it matches render.geometry within 1px. assert DS.barGapRatio.min <= barGap/avgBarWidth <= DS.barGapRatio.max. assert each plotMargins side within DS.margin range (±2px). assert aspectRatio ∈ DS.allowedAspectRatios (±0.02). assert tickLabelCount <= DS.maxTicks. assert title/subtitle/axis/legend y-origins ≡ 0 (mod gridUnit) within 1px (baseline grid snap).
  - threshold: all geometry metrics within DS ranges; baselines snap to grid; declared geometry matches re-parsed geometry
  - gameability: Token membership says nothing about layout; a crammed/misaligned chart with legal tokens previously passed. These are numeric range checks against DS-declared bounds — no aesthetic judgment, only 'inside the allowed envelope'. Independent re-parse blocks a faked geometry block.
- **VAR-13-cross-variant-equivalence** [blocker, consistency] — light, dark, and print_grayscale render the SAME data, encoding, claims, ordering, and bar heights — differing only in color/contrast tokens.
  - compute: assert variants.light.dataHash == variants.dark.dataHash == variants.print_grayscale.dataHash. For each variant SVG: parse bars; assert identical bar count, identical rowId ordering, identical bar-height pixel values (within 1px), and identical claim/title/callout text (string equality) across all three. Diff the SVG element trees; assert the only differences are fill/stroke/background token literals (structural node set identical).
  - threshold: identical dataHash, geometry (±1px), text, and structure across all three variants; only color tokens differ
  - gameability: Each variant could independently pass its own contrast check while being a different/degraded chart. Hash + structural diff forces them to be the same artifact reskinned, not three separate charts.
- **A11Y-14-contrast** [blocker, accessibility] — Every text/background and adjacent-bar pair meets WCAG contrast in every variant; the gate recomputes ratios from actual rendered colors.
  - compute: For every render.elements text node vs its background, recompute WCAG 2.x contrast ratio from the resolved token hex (per variant). assert text >= 4.5 (>=3.0 for large text >=24px or 18.7px bold), non-text graphic/adjacent-bar boundaries >= 3.0. Recompute independently rather than trusting qa.contrastReport; assert reported ratio matches recomputed within 0.05.
  - threshold: all text >=4.5 (large >=3.0), non-text >=3.0 in light/dark/print_grayscale; reported ratios match recomputed
  - gameability: qa.contrastReport could be fabricated. The gate recomputes from the actual SVG colors per variant, so a false report fails the match assertion.
- **A11Y-15-colorblind-separation** [blocker, accessibility] — Adjacent series/bars remain distinguishable under deuteranopia/protanopia/tritanopia, and distinguishability never relies on hue alone.
  - compute: For each adjacent rendered color pair, simulate each CVD type (Brettel/Viénot matrix) and compute CIEDE2000 deltaE. assert minAdjacentDeltaE >= 11.0 for every CVD type (recomputed by the gate, not trusted from qa.colorBlindSim). assert qa.colorBlindSim.nonColorEncodingPresent==true AND verify in SVG: each bar has a redundant encoding (direct value label OR distinct pattern fill OR text category adjacent).
  - threshold: recomputed min CVD deltaE >= 11.0 for all three CVD types AND a redundant non-color encoding present
  - gameability: deltaE is deterministic but had NO numeric threshold in the draft; the 'garish/on-brand palette' judgment is REMOVED (covered structurally by BRAND-10/11). A concrete >=11.0 threshold plus a mandatory non-hue encoding closes the 'distinguished by hue alone' failure.
- **FACE-16-disclosure-furniture** [blocker, standards] — Source line, as-of/data-vintage date, units, currency, and brand logo/wordmark are VISIBLE on the chart face (not only in metadata), and the logo respects clear-space.
  - compute: assert SVG contains text nodes whose normalized content includes disclaimers.asOfDate (formatted), disclaimers.units, and disclaimers.currency, AND a node with role 'sourceLine' whose text references provenance sourceSystem(s). assert exactly one element role 'logo' exists; compute its bounding box and the nearest neighbor distance; assert clearSpace = min distance to any other visible node >= DS.logoClearSpace (in px, derived from logo height). assert all furniture nodes are within plotArea-or-canvas bounds and have contrast passing A11Y-14.
  - threshold: as-of date, units, currency, source line, and logo all present as on-face visible nodes; logo clear-space >= DS.logoClearSpace
  - gameability: Furniture buried only in the metadata object previously passed the whole gate while failing basic financial-disclosure presentation. Requiring matching visible text nodes + a measured logo clear-space box makes presence-on-face computable.
- **SRC-17-provenance-resolves** [blocker, reproducibility] — Every provenance.sourceLocator resolves to a real cell/row/endpoint in the gate-supplied source file and yields the declared sourceValue.
  - compute: Load the source file pinned by checksum.sourceFileHash (gate-supplied; assert sha256(sourceFile)==sourceFileHash). For each provenance entry, parse sourceLocator (cell ref / JSON path / query result key) and read the value at that location; assert readValue == provenance.sourceValue (exact for ints, within 1e-9 for floats). assert every data.rows[rowId] has a provenance entry and vice versa (bijective).
  - threshold: 100% of provenance locators resolve in the real source and match sourceValue; provenance<->rows bijection complete
  - gameability: This is THE check that defeats self-consistent fabrication: a model can invent values + matching provenance + matching internal hashes, but it cannot make a fabricated sourceLocator resolve inside the gate-supplied real source file to the claimed value. Requires the gate harness to actually hold the real source artifact.
- **REPRO-18-hash-rerun** [blocker, reproducibility] — Re-running the pinned builder on the pinned source regenerates byte-identical dataHash and specHash.
  - compute: In the gate harness, run builderVersion against sourceFile (hash-verified). Recompute the canonicalized data and spec; assert sha256(rebuiltData)==checksum.dataHash AND sha256(rebuiltSpec)==checksum.specHash. assert builderVersion is a pinned, resolvable build identifier (commit/tag present in the build registry).
  - threshold: rebuilt dataHash and specHash byte-match the artifact's; builderVersion resolves to a pinned build
  - gameability: Ties the artifact to deterministic code over a real source. If the pipeline is non-deterministic or the hashes were hand-authored, the rerun diverges and fails. Depends on SRC-17 supplying the real source.
- **SORT-19-ordering-contract** [major, consistency] — Rendered bar order matches the declared sort contract (no taste-based reordering to flatter a narrative).
  - compute: Compute the expected rendered order from encoding.sort (by field + order, or declaredCategoryOrder from completeness.expectedCategories). Read actual left-to-right bar order from render.geometry.bars x-positions. assert actualOrder == expectedOrder.
  - threshold: rendered left-to-right order exactly equals the sort-contract order
  - gameability: Reordering bars to put a strong category first is an editorial framing trick; making sort an explicit contract and checking the rendered x-order against it removes the judgment and the trick.
- **MISS-20-missing-data-not-zerofilled** [blocker, correctness] — Genuinely missing data (valueRaw==null) is rendered as an explicit gap/marker, never silently drawn as a zero-height or interpolated bar.
  - compute: For each data.rows[i] with valueRaw==null: assert either no bar element exists for rowId AND a 'no data' marker node is present for that category, OR a bar exists only if an impute transform is in its transformChain AND disclaimers/notes flag imputation. assert no null-valueRaw row is rendered as a height-0 bar without a no-data marker.
  - threshold: every null row is shown as an explicit no-data marker or a disclosed imputation; zero silent zero-fills
  - gameability: Zero-filling missing quarters fabricates a downturn or flat reading. Tying null handling to either an explicit marker or a DISCLOSED impute transform makes silent fabrication detectable.

## Finance / Accounting — Quarter-close reconciliation workbook (workbook + journal-entry batch + sign-off memo) posted by a controller into the general ledger. Acceptance must satisfy GAAP/IFRS recognit

**Ship gate:** SHIP only if ALL of the following hold, evaluated deterministically in plain JS over the enriched artifact (+ brandTokenAllowlist fixture): (1) ZERO blocker-severity checks fail — blockers are C01, C02, C03, C04, C05, C06, C08, C09, C10, C11, C12, C13, C14, C15, C16, C18, C19; (2) ZERO major-severity checks fail — C07, C17, C20, C21 (no majors are waivable for a GL-posting artifact); (3) schema completeness: all required fields present and type-valid, including renderedCells, declaredTotals, chartOfAccountsInScope, and brandTokenAllowlist (absence of any = automatic blocker fail); (4) the three-way tie of C14 (rendered headline == declaredTotals == recomputed component sums) holds to 0.00. Any single failing check blocks the post. There is no human-judge fallback and no LLM scoring anywhere in the gate — every predicate is an arithmetic comparison, set-membership test, regex match, date comparison, or WCAG-luminance computation.

**Expected frontier pass rate:** Estimated single-pass frontier pass rate: ~8-15%. Reasoning: a frontier model will reliably satisfy the arithmetic recognizability checks it is 'looking for' — per-row foot/cross-foot (C01), debits==credits (C02), maker-checker distinctness (C08/C09) — because those are the obvious finance checks. The kill zone is the cross-representation and accessibility coupling that models systematically neglect: (a) C13/C14 require the rendered strings to re-derive exactly from rawValue AND tie to declaredTotals AND tie to recomputed sums — three independent representations that must agree to the cent; models almost always let the rendered headline drift or omit renderedCells entirely. (b) C12 requires a SINGLE negative style and separator across every rendered cell; mixed paren/minus is near-universal. (c) C18/C19 require a genuinely distinct, contrast-verified dark variant plus a redundant non-color encoding for every semantic state — finance outputs are reflexively red/green and ship cloned or low-contrast dark modes. (d) C16 requires period-token + basis-token + non-boilerplate certification text from a known signer — models emit generic or templated certifications. (e) C17 demands every color resolve to a fixture allowlist. (f) C03/C04 referential integrity + full-scope completeness against an independently declared chart catch the cherry-picked-subset and orphaned-item patterns. The probability of clearing ALL ~21 gates (17 blockers + 4 majors) on one pass — given each of the ~6 hard, rarely-co-satisfied gates has maybe a 50-70% independent pass chance and the arithmetic gates ~85-95% — multiplies down to roughly 0.10. Well below 100%, as required.

### Enriched artifact schema
- `period` (object {fiscalYear:int, quarter:enum[Q1..Q4], periodStart:ISO8601 date, periodEnd:ISO8601 date, closeType:enum[hard,soft]}, required) — Every number must be scoped to an explicit close period; reconciliations and cutoff tests are meaningless without a defined boundary. periodEnd must be > periodStart and fall inside fiscalYear.
- `declaredTotals` (object {grandTotalGL:decimal(18,2), grandTotalSource:decimal(18,2), grandTotalReconcilingItems:decimal(18,2), grandTotalReconciledDifference:decimal(18,2)}, required) — The draft C01 referenced an undefined declaredGrandTotalGL. The workbook must DECLARE its own headline totals so the recompute-from-components check has a canonical target and the rendered headline numbers (most visible cells) have a single source of truth.
- `accounts[]` (array of {glAccountNumber:string, accountName:string, accountType:enum[asset,liability,equity,revenue,expense], glBalance:decimal(18,2), subledgerOrSourceBalance:decimal(18,2), sourceId:string, reconcilingItemIds:string[], reconciledDifference:decimal(18,2), tolerance:decimal(18,2), preparerNote:string, inScope:bool}, required) — Core of a reconciliation: GL vs independent source per account, with explained gap. Must cover the FULL in-scope chart of accounts (declared via chartOfAccountsInScope), not a toy subset. sourceId binds the balance to a provenance record; reconcilingItemIds bind to the items array (referential integrity).
- `chartOfAccountsInScope[]` (array of {glAccountNumber:string, accountName:string, accountType:enum[asset,liability,equity,revenue,expense], requiresRecon:bool}, required) — Declares the full population so completeness (no missing accounts) is checkable. Without a declared universe, a model can ship a cherry-picked subset and look complete.
- `reconcilingItems[]` (array of {id:string, accountNumber:string, description:string, amount:decimal(18,2), ageDays:int, ageBucket:enum[0-30,31-60,61-90,90+], sourceDocRef:string, expectedClearDate:ISO8601 date, status:enum[open,cleared,written-off], approvedBy:string|null, nonColorStatusLabel:string}, required) — Unexplained differences are the audit red flag. Each item must be aged, sourced, dispositioned; stale items need approval. nonColorStatusLabel enforces WCAG 1.4.1 (status not color-only).
- `journalEntries[]` (array of {jeId:string, postingDate:ISO8601 date, lines[]:{accountNumber:string, debit:decimal(18,2), credit:decimal(18,2), entity:string, costCenter:string}, description:string, supportRef:string, preparedBy:string, approvedBy:string, isManual:bool, isTopside:bool, reversalOf:string|null}, required) — What actually posts to the GL. Must balance per entry, carry maker-checker (preparedBy != approvedBy), and flag manual/topside entries (PCAOB high-risk areas) which require explicit approval + support.
- `sources[]` (array of {sourceId:string, systemName:string, extractTimestamp:ISO8601 datetime, extractMethod:enum[api,export,manual], rowCount:int, controlTotal:decimal(18,2), hash:sha256, ownerSystemOfRecord:bool}, required) — Provenance. Every balance must trace to a named source extract with a control total and SHA-256 hash so the reconciliation is reproducible and tamper-evident. extractTimestamp must be on/after periodEnd.
- `assumptions[]` (array of {id:string, statement:string, basis:string, fxRateUsed:decimal|null, fxRateSource:string|null, estimateMethod:string|null}, required) — Accruals, allowances, FX, eliminations rest on assumptions that must be stated and sourced (substantiation), not buried in cell formulas.
- `fx` (object {functionalCurrency:string, presentationCurrency:string, rates[]:{currencyPair:string, rate:decimal(12,6), rateType:enum[spot,average,closing], rateSource:string, rateDate:ISO8601 date}}, required) — Multi-currency consolidation (IAS 21 / ASC 830) requires exact rate, type, source, date for every translation. Every non-functional currency appearing in JE lines/accounts must have a matching rate row.
- `controls` (object {preparedBy:{name,role,timestamp}, reviewedBy:{name,role,timestamp}, approvedBy:{name,role,timestamp}, segregationOk:bool, materialityThreshold:decimal(18,2), performanceMateriality:decimal(18,2)}, required) — SOX/COSO require documented maker-checker with segregation of duties and explicit materiality basis. The three named persons must be distinct; performanceMateriality < materialityThreshold; timestamps ordered prepared <= reviewed <= approved.
- `disclaimersAndCertifications` (object {basisOfPreparation:enum[US-GAAP,IFRS], estimateDisclaimer:string, scopeLimitations:string[], certificationStatement:string, certifiedBy:string, certificationDate:ISO8601 date}, required) — Legal/claims substantiation: the package asserts books are fairly stated; that assertion needs a basis-of-prep statement, estimate caveats, named certifier (SOX 302 spirit). Free-text fields get a non-empty + non-boilerplate + must-reference-period gate (see C16).
- `renderSpec` (object {tokens:{colorRoles:map<string,hex>}, lightVariant:{bg:hex,text:hex,...roleColors}, darkVariant:{bg:hex,text:hex,...roleColors}, focusStateSpec:{indicator:string, color:hex, widthPx:number}, numberFormat:{thousandsSep:char, decimalSep:char, negativeStyle:enum[paren,minus], currencySymbol:string, currencySymbolPosition:enum[before,after]}, locale:string}, required) — The workbook ships as a rendered/exported report. Brand tokens, light+dark variants, focus states, and a single canonical number format must be declared AND verified (C12–C18) so accessibility/consistency are deterministically checkable rather than delegated to a human.
- `brandTokenAllowlist[]` (array of {role:string, hex:hex} (the approved enterprise palette/token registry, supplied as fixture), required) — Without an approved palette to resolve against, renderSpec.tokens.colorRoles can be arbitrary. This makes brand-token conformance (C17) deterministic.
- `renderedCells[]` (array of {ref:string, rawValue:decimal, renderedString:string, semanticState:enum[normal,negative,over-tolerance,stale,status-open,status-cleared,status-written-off]|null}, required) — Closes the 'totals-as-rendered never reconciled to totals-as-computed' and 'number-format incoherence' holes. Each rendered string is checked against renderSpec.numberFormat (C12) AND against round(rawValue,2) re-rendered (C13). Headline total cells must reconcile to declaredTotals (C14).
- `auditTrail[]` (array of {event:string, actor:string, timestamp:ISO8601 datetime, before:any, after:any}, required) — Reproducibility/traceability: every override, manual adjustment, re-extract logged immutably for the auditor to re-walk. Every JE flagged isManual or isTopside, and every reconcilingItem with status 'written-off', must have a corresponding auditTrail event (C11).

### Checks (21)
- **C01-foot-cross-foot** [blocker, correctness] — Every account row foots and cross-foots from its COMPONENTS, and column subtotals equal the DECLARED grand totals to the cent. The stated reconciledDifference is never trusted; it is recomputed.
  - compute: For each account a: sumItems = round(sum(item.amount for item where item.id in a.reconcilingItemIds), 2); assert round(a.glBalance - a.subledgerOrSourceBalance - sumItems, 2) === round(a.reconciledDifference, 2). Then assert round(sum(a.glBalance for inScope accounts),2) === round(declaredTotals.grandTotalGL,2); same for grandTotalSource, grandTotalReconcilingItems, grandTotalReconciledDifference. FAIL on any mismatch != 0.00.
  - threshold: 0 cents tolerance on every row and every declared total; 0 failing rows
  - gameability: Model can hardcode reconciledDifference so rows tie without items summing. Caught because we recompute from reconcilingItemIds and require the declared grand totals (now a schema field) to equal the recomputed column sums.
- **C02-debits-equal-credits** [blocker, correctness] — Every journal entry balances: sum of debits equals sum of credits per JE, and the whole batch nets to zero.
  - compute: For each je: assert round(sum(l.debit for l in je.lines),2) === round(sum(l.credit for l in je.lines),2). Assert no line has both debit>0 AND credit>0. Assert round(sum(all debits) - sum(all credits),2) === 0.00.
  - threshold: 0 cents per entry and batch-wide; 0 unbalanced entries
  - gameability: A single-sided line could net at batch level while an individual JE is unbalanced; per-entry assertion plus no-mixed-sign-line assertion prevents that.
- **C03-referential-integrity** [blocker, correctness] — All foreign keys resolve: every account.sourceId exists in sources[]; every id in account.reconcilingItemIds exists in reconcilingItems[]; every reconcilingItem.accountNumber and JE line.accountNumber exists in chartOfAccountsInScope[]; every JE reversalOf points to an existing jeId.
  - compute: Build sets of sourceIds, itemIds, jeIds, COA account numbers. For each reference, assert membership. Assert no reconcilingItem is orphaned (its id appears in exactly one account's reconcilingItemIds). FAIL on any dangling or orphaned reference.
  - threshold: 0 dangling references, 0 orphaned reconciling items
  - gameability: Decouples numbers that 'look right' from a graph that isn't wired; prevents items counted in totals but attached to no account.
- **C04-completeness-full-scope** [blocker, completeness] — Every chartOfAccountsInScope account with requiresRecon=true appears exactly once in accounts[] with inScope=true; no extra accounts outside the declared chart.
  - compute: reqSet = {a.glAccountNumber for a in chartOfAccountsInScope if a.requiresRecon}; reconSet = {a.glAccountNumber for a in accounts if a.inScope}. Assert reqSet ⊆ reconSet AND reconSet ⊆ {all COA account numbers} AND no duplicate glAccountNumber in accounts[]. FAIL if any required account missing or any reconciled account not in COA.
  - threshold: 0 missing required accounts, 0 out-of-chart accounts, 0 duplicates
  - gameability: Stops cherry-picking a tidy subset: the in-scope population is declared independently and must be fully covered.
- **C05-reconciling-item-aging** [blocker, correctness] — ageBucket is consistent with ageDays, and ageDays is consistent with periodEnd minus the item's basis date; every open item older than 90 days is approved.
  - compute: For each item: assert bucket(item.ageDays) === item.ageBucket where bucket maps 0-30/31-60/61-90/90+. Assert item.ageDays >= 0. For each item with status='open' and ageDays>90: assert item.approvedBy is a non-empty string. FAIL otherwise.
  - threshold: 0 bucket mismatches, 0 unapproved stale (>90d open) items
  - gameability: A model can mislabel an old item into a young bucket to dodge the approval requirement; recomputing the bucket from ageDays closes that.
- **C06-tolerance-gate** [blocker, correctness] — Every account's absolute reconciled difference is within its declared tolerance, and tolerance never silently exceeds performance materiality.
  - compute: For each account: assert abs(round(a.reconciledDifference,2)) <= round(a.tolerance,2). Assert round(a.tolerance,2) <= round(controls.performanceMateriality,2). FAIL on any account whose unexplained difference exceeds tolerance or whose tolerance exceeds performance materiality.
  - threshold: 0 accounts over tolerance; 0 tolerances above performance materiality
  - gameability: A model could set a giant per-account tolerance to swallow a real difference; capping tolerance at performance materiality removes that escape.
- **C07-materiality-ordering** [major, correctness] — Materiality basis is coherent: performanceMateriality strictly less than materialityThreshold, both positive.
  - compute: assert controls.materialityThreshold > 0 AND controls.performanceMateriality > 0 AND controls.performanceMateriality < controls.materialityThreshold. FAIL otherwise.
  - threshold: strict inequality holds; both > 0
  - gameability: Equal or inverted thresholds make the tolerance gate meaningless; strict ordering enforced.
- **C08-segregation-of-duties** [blocker, correctness] — Maker-checker is real: preparedBy, reviewedBy, approvedBy are three distinct named persons; timestamps ordered; segregationOk asserted true and consistent.
  - compute: names = [controls.preparedBy.name, controls.reviewedBy.name, controls.approvedBy.name]; assert all non-empty AND new Set(names).size === 3. Assert prepared.timestamp <= reviewed.timestamp <= approved.timestamp (valid ISO, monotonic). Assert controls.segregationOk === true. FAIL otherwise.
  - threshold: 3 distinct signers, monotonic timestamps, segregationOk true
  - gameability: segregationOk:true alone is trivially set; we independently verify distinct names and timestamp ordering rather than trusting the flag.
- **C09-je-maker-checker** [blocker, correctness] — Each journal entry has distinct preparedBy and approvedBy (both non-empty); every manual OR topside entry additionally has a non-empty supportRef.
  - compute: For each je: assert je.preparedBy && je.approvedBy && je.preparedBy !== je.approvedBy. If je.isManual || je.isTopside: assert je.supportRef is non-empty AND je.approvedBy is non-empty. FAIL otherwise.
  - threshold: 0 JEs with same maker/checker; 0 manual/topside JEs missing support
  - gameability: PCAOB treats manual/topside entries as high-risk; requiring support+approver on exactly those entries prevents unsupported overrides slipping in.
- **C10-source-provenance** [blocker, reproducibility] — Every source extract is tamper-evident and timely: hash is a valid SHA-256, extractTimestamp on/after periodEnd, controlTotal present, and exactly one ownerSystemOfRecord source per account.
  - compute: For each source: assert /^[0-9a-f]{64}$/.test(s.hash); assert s.rowCount >= 0; assert Date(s.extractTimestamp) >= Date(period.periodEnd). For each account: assert the source referenced by a.sourceId exists and has ownerSystemOfRecord===true. FAIL otherwise.
  - threshold: 0 invalid hashes, 0 pre-period extracts, every account backed by a system-of-record source
  - gameability: A non-hex/short hash or a stale pre-period extract makes provenance unverifiable; regex + date floor catch placeholders like 'TBD' or 'abc123'.
- **C11-audit-trail-coverage** [blocker, reproducibility] — Every high-risk mutation is logged: each manual or topside JE, and each written-off reconciling item, has at least one matching auditTrail event referencing its id.
  - compute: For each je where isManual||isTopside: assert exists e in auditTrail with e.after/e.before/e.event string containing je.jeId. For each item where status==='written-off': assert exists e in auditTrail referencing item.id. Assert every auditTrail event has valid ISO timestamp, non-empty actor, non-empty event. FAIL on any uncovered mutation.
  - threshold: 100% of manual/topside JEs and written-off items have an audit event; 0 malformed events
  - gameability: Without id-level coverage a model logs one generic event and claims a trail; we require an event per high-risk object id.
- **C12-number-format-canonical** [blocker, consistency] — Every rendered numeric cell obeys the single canonical numberFormat — one negative style, one thousands sep, one decimal sep, one currency-symbol position — across the entire workbook.
  - compute: Derive pattern from renderSpec.numberFormat. For each cell in renderedCells: regex-assert cell.renderedString matches the canonical pattern; if value<0 assert it uses renderSpec.negativeStyle (paren => /^\(.*\)$/, minus => /^-/) and NOT the other. Assert count(distinct negative styles observed across negatives) === 1. Assert observed thousandsSep/decimalSep/symbol-position are uniform. FAIL on any deviation.
  - threshold: 100% of rendered cells match the single canonical pattern; exactly 1 negative style observed
  - gameability: C01/C02 operate on numeric values, not strings, so mixed -100 vs (100), separator drift, and symbol drift pass arithmetic while looking broken; this checks the rendered strings directly.
- **C13-render-equals-value** [blocker, correctness] — Each rendered string equals its own rawValue re-rendered under renderSpec — no render-layer rounding/format bug.
  - compute: For each cell in renderedCells: expected = renderNumber(round(cell.rawValue,2), renderSpec.numberFormat); assert cell.renderedString === expected. FAIL on any mismatch.
  - threshold: 0 cells where renderedString != deterministic render of rawValue
  - gameability: A cell can pass C12's pattern yet show the wrong number; re-rendering rawValue and comparing exact strings catches a value/label divergence.
- **C14-headline-totals-tie** [blocker, correctness] — The rendered headline total cells equal the declaredTotals which equal the recomputed component sums — render, declaration, and math all agree.
  - compute: Identify renderedCells whose ref maps to grandTotalGL/grandTotalSource/grandTotalReconcilingItems/grandTotalReconciledDifference. For each: assert round(cell.rawValue,2) === round(declaredTotals[corresponding],2) AND cell.renderedString === renderNumber(declaredTotals[corresponding], renderSpec). Combined with C01 (declared===recomputed), this closes the loop. FAIL on any gap.
  - threshold: 0 divergence between rendered headline, declared total, and computed sum
  - gameability: The most visible numbers can disagree with the math that passed; this ties the three representations together to the cent.
- **C15-locale-format-coherence** [blocker, consistency] — renderSpec.locale and numberFormat are internally consistent and non-empty: valid BCP-47 tag, thousandsSep != decimalSep, both non-empty single chars, currencySymbolPosition in {before,after}, negativeStyle in {paren,minus}.
  - compute: assert /^[A-Za-z]{2,3}(-[A-Za-z0-9]{2,8})*$/.test(renderSpec.locale); assert numberFormat.thousandsSep && numberFormat.decimalSep && numberFormat.thousandsSep !== numberFormat.decimalSep; assert ['before','after'].includes(currencySymbolPosition); assert ['paren','minus'].includes(negativeStyle); assert currencySymbol non-empty. FAIL on any contradiction or empty field.
  - threshold: all fields present, valid, non-contradictory
  - gameability: Draft schema allowed locale:'en-US' with decimalSep:',' (self-contradictory); separator-inequality + BCP-47 validation reject it.
- **C16-disclosure-content-gate** [blocker, completeness] — Certification and disclosure free-text are present, non-boilerplate, and scoped to THIS period — not blank, lorem-ipsum, or generic template.
  - compute: For estimateDisclaimer, certificationStatement, each scopeLimitations entry: assert length >= 40 chars; assert !/lorem ipsum|TBD|TODO|placeholder|xxx|\[.*\]/i.test(text). Assert certificationStatement contains the period token (e.g. `Q{quarter} {fiscalYear}` OR periodEnd year) AND basisOfPreparation value. Assert certifiedBy non-empty AND === one of the three control signers. Assert certificationDate >= controls.approvedBy.timestamp date. FAIL otherwise.
  - threshold: 0 blank/boilerplate disclosures; certification references this period + basis; certifier is a known signer
  - gameability: Free-text fields can be empty/templated/contradicting the numbers; period-token + basis-token + length + boilerplate-regex + signer-membership make the assertion concrete without an LLM.
- **C17-brand-token-allowlist** [major, brand] — Every color in renderSpec.tokens.colorRoles and in light/dark variants resolves to an approved brandTokenAllowlist entry; no arbitrary/placeholder hexes.
  - compute: allow = {hex.toLowerCase() for entry in brandTokenAllowlist}. For each hex in tokens.colorRoles values + lightVariant + darkVariant: assert /^#[0-9a-f]{6}$/i.test(hex) AND allow.has(hex.toLowerCase()). Flag default placeholders (#000000,#ffffff,#ff0000) as FAIL unless explicitly present in allowlist. FAIL on any non-allowlisted color.
  - threshold: 100% of rendered colors are allowlist members; 0 placeholder defaults
  - gameability: tokens.colorRoles can map to arbitrary off-brand colors; resolving against a fixture allowlist makes brand identity deterministically enforceable.
- **C18-contrast-and-dark-mode** [blocker, accessibility] — Light and dark variants are real, distinct, and both WCAG AA conformant; focus state is specified with a visible, sufficiently-contrasting indicator.
  - compute: assert JSON.stringify(lightVariant) !== JSON.stringify(darkVariant). For each variant, for each text-on-bg role pairing: compute WCAG contrast ratio via relative luminance; assert >= 4.5 for body text, >= 3.0 for large/UI text. Assert focusStateSpec.indicator non-empty, focusStateSpec.widthPx >= 2, and contrast(focusStateSpec.color, adjacent bg) >= 3.0 in BOTH variants. FAIL if any variant missing, byte-identical, or below ratio.
  - threshold: variants distinct; all text pairs >= 4.5 (>=3.0 large); focus indicator present and >= 3.0
  - gameability: darkVariant can be a clone of light (fake dark mode) or low-contrast; byte-inequality + computed contrast ratios catch both.
- **C19-no-color-only-meaning** [blocker, accessibility] — Color is never the sole carrier of meaning (WCAG 1.4.1): every semantic state has a redundant non-color encoding.
  - compute: For each renderedCell with non-null semanticState (negative/over-tolerance/stale/status-*): assert a non-color indicator exists — negatives carry sign or parens (per negativeStyle), reconcilingItems carry nonColorStatusLabel matching status, over-tolerance/stale cells carry a textual flag in renderedString or an associated label field. Enumerate states {reconciledDifference>tolerance, ageBucket==='90+', status open/cleared/written-off, value<0}; assert each has a declared non-color indicator. FAIL if any state is color-only.
  - threshold: 0 states encoded by color alone
  - gameability: Classic finance red/green debit/credit + aging coding fails colorblind users; requiring a redundant label/sign/icon per state closes it.
- **C20-fx-coverage** [major, correctness] — Every non-functional currency used anywhere (account or JE line) has a matching, dated fx rate row of the correct type; functional==presentation needs no translation but must be declared.
  - compute: usedCurrencies = currencies inferred from JE line entity/currency + accounts not in functionalCurrency. For each used currency != fx.functionalCurrency: assert exists rate in fx.rates with currencyPair containing that currency, valid rate>0, rateType in {spot,average,closing}, non-empty rateSource, rateDate within [periodStart, periodEnd]. FAIL on any uncovered currency or out-of-period rate.
  - threshold: 100% of used non-functional currencies have an in-period, sourced rate
  - gameability: IAS 21/ASC 830 require the exact rate/type/source/date; a missing or undated rate (or a rate dated outside the period) is a translation error the arithmetic checks miss.
- **C21-period-validity** [major, correctness] — The close period is well-formed and all dated objects fall in a defensible window relative to it.
  - compute: assert Date(period.periodStart) < Date(period.periodEnd); assert periodEnd year === fiscalYear (or fiscalYear boundary rule); assert quarter in [Q1..Q4]; assert every je.postingDate within [periodStart, periodEnd] OR flagged in auditTrail as adjusting; assert no expectedClearDate precedes periodEnd for status='open' items by more than 0 (i.e. expectedClearDate >= periodEnd). FAIL on malformed period or postings outside the period without trail.
  - threshold: valid period; 0 untracked out-of-period postings
  - gameability: Cutoff manipulation (posting into a closed period) is a core close risk; bounding postingDate to the period or an audit-logged adjustment catches it.

## Shared primitive library
- **selfReportRecomputeMatch** `(reportedValue: number, recompute: () => number, tolerance: number) -> {pass: boolean, computed: number, delta: number}` — Independently recompute the metric from raw inputs (never trust the artifact's self-reported number); FAIL if abs(computed - reported) > tolerance. Seen in C01 (wcagRatio within 0.05), C04 (apcaLc within 1.0), C11 (focus contrasts), C12 (CVD deltaE within 0.5) -- the universal anti-fabrication guard pairing any computed metric with the claimed one.
- **contrastRatioWCAG** `(fg: sRGB, bg: sRGB) -> ratio: number` — Linearize each channel (c<=0.03928 ? c/12.92 : ((c+0.055)/1.055)^2.4), L=0.2126R+0.7152G+0.0722B, ratio=(max(L1,L2)+0.05)/(min(L1,L2)+0.05); compare against role-specific floor (4.5 body / 3.0 nonText+large). Core of C01/C02/C03/C11.
- **thresholdTableLookup** `(keyDims: object, table: Map<key,minimum>, measured: number) -> {pass: boolean, required: number}` — Index a bundled, contract-owned data table by the artifact's own resolved dimensions (e.g. px size + weight), read the required minimum, PASS if measured >= that minimum. Replaces a flat magic-number judgment with deterministic data the contract owns (C04 APCA font matrix); same pattern any time a per-context floor must be principled not arbitrary.
- **deterministicSimulationRecompute** `(inputs: T[], publishedTransform: (T)=>T, pairwiseMetric: (T,T)=>number, floor: number) -> {pass: boolean, min: number}` — Apply a published deterministic transform (e.g. Vienot/Brettel CVD matrices), compute the pairwise distance metric (CIEDE2000) across transformed items, PASS if min >= contract-owned floor. C12 -- generalizes to any 'simulate a stressor, require separation survives' check.
- **canonicalSourceTrace** `(declaredSourcePath: string, citedValues: Map<id,value>, parsedFromFile: Map<id,value>) -> {pass: boolean, mismatches: id[]}` — Require the artifact to declare ONE canonical source, parse that file at the cited path, and assert artifact values equal file values exactly (case-insensitive); FAIL if source unset, unresolvable, or any value mismatches. C05 -- the general claim-to-source grounding primitive (each asserted value must trace verbatim to its declared origin).
- **typedSchemaConformance** `(items: object[], requiredFields: {name,validator}[]) -> {violations: number}` — For every item assert each required field is present and passes its validator (enum membership, non-empty, min length, layer/category in allowed set); count violations, threshold 0. C06 -- structural lint applied to every record.
- **forbiddenLiteralLint** `(items: object[], scope: predicate, literalPattern: RegExp, allowedRefPattern: RegExp) -> {violations: number}` — Within scoped items, FAIL any value matching a forbidden raw-literal pattern (hex/rgb/hsl) that is not an allowed reference string; enforces indirection. C07 first half -- generalizes to 'no hardcoded literals where an alias/constant is required'.
- **referenceGraphResolve** `(aliases: Map<id, refId|value>) -> {danglingRefs: id[], cycles: id[][]}` — Build the alias/dependency graph, resolve each reference to a concrete value, FAIL on any ref to a missing id (dangling) or any cycle (DFS back-edge). C07 second half -- integrity of any pointer/alias/dependency network.
- **antiCollapseDistinctness** `(families: Map<roleFamily, resolvedValue[]>, minDistinct: number, opposedPairs: [role,role][], collisionMetric, collisionFloor) -> {pass: boolean}` — Count distinct resolved values across role families (require >= minDistinct) AND assert no two semantically-opposed roles resolve to the same value (distance < floor = collision). C08 -- defeats 'launder everything to one value' / mode-collapse where mechanical referencing would otherwise pass.
- **perceptibleDelta** `(variants: Map<state, resolvedValue>, deltaMetric, jndThreshold: number) -> {pass: boolean}` — For each pair of states that must differ, compute the perceptual/semantic distance (CIEDE2000>=2.0 JND) and FAIL if below the just-noticeable-difference threshold; bounded-band variants (disabled Lc in [30,60]) get a range check. C09 -- kills 'emit N near-identical values' to satisfy a count.
- **uniqueNonGenericCue** `(items: object[], fields: string[], allowedSet?: Set, bannedGeneric: Set) -> {pass: boolean}` — For each named field assert value is in the allowed set (if any), pairwise-unique across items, and not in the banned-generic set ('Status','Item',''); FAIL otherwise. C10 -- naming/label convention guard that blocks clone-and-rename dodges.
- **coverageSuperset** `(declaredSet: Set, machineExtractedRequiredSet: Set) -> {missing: id[]}` — Machine-extract the full set of required items from the real sources, then assert declaredSet superset-of requiredSet; FAIL listing any required item the artifact omitted. C17 + C11 appliedToRoles -- defeats 'declare only the easy/passing subset'.
- **worstCaseResolution** `(value: ColorOrRange, assumedContext, compositeFn) -> opaqueResolved` — Resolve translucent/gradient/range inputs to the single worst-case concrete value (alpha-composite onto declared backdrop, pick worst gradient stop) before evaluating any threshold; removes the flattering-assumption escape hatch. C16 -- the adversarial-resolution rule feeding C01/C02/C11.
- **diffAgainstLiveRepo** `(artifactSet: Set, liveSet: Set) -> {removed: id[], renamed: id[], breakingButUndeclared: id[]}` — Extract live identifiers from the actual repo (regex over export keys / --var names), diff against the artifact set, and reject a declared 'no-breaking-change' when live consumers reference removed/renamed ids. C13 -- grounds reproducibility/changelog claims in the real prior state, not a greenfield assertion.
- **labelJustifiedByResolvedAttr** `(item: object, claimedClass: string, resolvedAttrs: object, rule: predicate) -> {pass: boolean}` — Cross-check that an item's self-assigned class is justified by its resolved underlying attributes (e.g. role='large' requires resolved size>=24px OR >=18.66px+weight>=700 from the real type scale); FAIL on mislabel. C03 cross-check -- blocks relabeling to dodge a stricter threshold.
